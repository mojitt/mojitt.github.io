## HTTP 쿠키 (Cookie)
쿠키(Cookie)란?
- 서버가 브라우저에 저장하는 텍스트 형태의 작은 데이터 파일.
- 브라우저는 같은 서버에 요청을 보낼 때 저장된 쿠키를 자동으로 HTTP 헤더에 포함하여 전송.
- → 서버는 쿠키를 통해 이전 방문자를 식별하거나 상태를 유지할 수 있음.

- 주요 용도: 로그인 상태 유지 / 팝업 "오늘 하루 보이지 않기" / 장바구니 / 사용자 설정 저장

#### 쿠키의 특징
[ 쿠키 특징 ]

| 항목 | 설명 |
|---|---|
| 저장 위치 | 브라우저 (클라이언트) |
| 저장 형식 | `key=value` 문자열. 여러 개는 `;` 로 구분. |
| 용량 제한 | 도메인당 약 4KB 이하, 최대 20~50개 |
| 유효기간 | `expires` 또는 `max-age` 로 설정. 미설정 시 브라우저 종료와 함께 삭제 (세션 쿠키). |
| 자동 전송 | 같은 도메인의 모든 HTTP 요청에 자동으로 포함되어 서버로 전송됨. |
| 접근 방법 | JS: `document.cookie` / 서버: HTTP 헤더 (`Set-Cookie`) |

#### 쿠키 vs 웹 스토리지 (localStorage / sessionStorage)
[ 쿠키 vs 웹 스토리지 비교 ]

| 구분 | 쿠키 (Cookie) | localStorage | sessionStorage |
|---|---|---|---|
| 용량 | 약 4KB | 약 5~10MB | 약 5~10MB |
| 만료 | 직접 설정 가능 | 직접 삭제 전까지 영구 | 탭(세션) 종료 시 삭제 |
| 서버 전송 | 자동으로 HTTP 요청에 포함 | 전송 안 됨 | 전송 안 됨 |
| 접근 | JS + 서버 양쪽 | JS만 | JS만 |
| 주요 용도 | 로그인 세션, 팝업 설정 | 사용자 설정, 앱 상태 저장 | 임시 데이터 (탭 내 유지) |

#### document.cookie — 쿠키 읽기 / 쓰기
- document.cookie는 일반적인 문자열 프로퍼티처럼 보이지만 동작 방식이 다름.

쓰기: 할당 시 기존 쿠키를 덮어쓰지 않고 해당 key만 추가 또는 수정.
- document.cookie = "key=value; expires=날짜; path=/"  

읽기: 현재 페이지에서 접근 가능한 모든 쿠키를 하나의 문자열로 반환.
- "key1=value1; key2=value2; key3=value3" 형태 → 파싱 필요.

삭제: 직접 삭제 API 없음. expires를 과거 날짜로 설정하면 즉시 삭제됨.  

[ 쿠키 옵션 (Attribute) ]

| 옵션 | 설명 | 예시 |
|---|---|---|
| `expires` | 쿠키 만료 날짜 (UTC 문자열). 미설정 시 세션 쿠키. | `expires=Fri, 31 Dec 2025 23:59:59 GMT` |
| `max-age` | 쿠키 유효 시간(초). `expires` 보다 우선 적용. | `max-age=86400` (1일) |
| `path` | 쿠키가 유효한 경로. 기본값은 현재 경로. `/` 로 설정 시 전체 사이트. | `path=/` |
| `domain` | 쿠키가 유효한 도메인. | `domain=example.com` |
| `secure` | HTTPS 연결에서만 전송. | `secure` |
| `HttpOnly` | JS에서 접근 불가. 서버에서만 설정 가능. XSS 공격 방어. | `HttpOnly` |
| `SameSite` | 크로스 사이트 요청 시 쿠키 전송 여부 제어. CSRF 방어. | `SameSite=Strict` / `Lax` / `None` |


```
// ── 쿠키 저장 (setCookie)
// key, value, 유효일수(day)를 받아 document.cookie에 저장
const setCookie = (key, value, day) => {
    const date = new Date();
    date.setDate(date.getDate() + day);          // 현재 날짜 + day일
    const expires = date.toUTCString();          // UTC 문자열로 변환
    document.cookie = `${key}=${value}; expires=${expires}; path=/`;
};

// ── 쿠키 읽기 (getCookie)
// document.cookie = "key1=val1; key2=val2" 형태의 문자열을 파싱
const getCookie = key => {
    const cookies = document.cookie.split('; ');  // '; '로 분리
    const found = cookies.find(c => c.startsWith(key + '='));
    return found ? found.split('=')[1] : null;    // 없으면 null
};

// ── 쿠키 삭제 (deleteCookie)
// expires를 과거로 설정하면 브라우저가 즉시 삭제
const deleteCookie = key => {
    document.cookie = `${key}=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/`;
};

// ── 전체 쿠키 객체로 변환
const getAllCookies = () => {
    if (!document.cookie) return {};
    return Object.fromEntries(
        document.cookie.split('; ').map(c => c.split('='))
    );
}; 
```

#### 팝업 쿠키 실전 — "오늘 하루 보이지 않기"
- 페이지 로드 시 popupClose 쿠키를 확인.
- 쿠키가 없거나 값이 'Y'가 아니면 팝업을 표시.
- 팝업의 "오늘 하루 보이지 않기" 체크 후 닫으면 popupClose=Y 쿠키를 오늘 자정까지 저장.
- → 다음 날 자정이 지나면 쿠키가 만료되어 다시 팝업이 표시됨.

```
const $popupOverlay = document.querySelector('#popupOverlay');
const togglePopup = isOpen => $popupOverlay.style.display = isOpen ? 'flex' : 'none';
const popupClose = getCookie('popupClose');
const $popupCloseBtn = document.querySelector('#popupCloseBtn');

if(popupClose !== 'Y') togglePopup(true);

$popupCloseBtn.addEventListener('click', e => {
    const checked = document.getElementById('popupCloseCheck').checked;
    if(checked) {
        const date = new Date();
        date.setHours(23,59,59,0);          // 자정
        const expires = date.toUTCString(); // UTC 문자열로 변환
        document.cookie = `popupClose=Y; expires=${expires}; path=/`;
    }
    togglePopup(false);
});

const $openPopupBtn = document.querySelector('#openPopupBtn');
$openPopupBtn.addEventListener('click', e => togglePopup(true));

const $resetPopupCookie = document.querySelector('#resetPopupCookie');
$resetPopupCookie.addEventListener('click', e => {
    deleteCookie('popupClose');
    location.reload();
});
```

#### 보안 — 쿠키 취약점과 방어 (중급)
- 쿠키는 클라이언트에 저장되기 때문에 보안에 취약할 수 있음.
- 민감한 정보(비밀번호, 주민번호 등)는 절대 쿠키에 저장하지 않음.

[ 쿠키 보안 옵션 ]

| 공격 유형 | 설명 | 방어 방법 |
|---|---|---|
| XSS (크로스 사이트 스크립팅) | 악성 스크립트로 `document.cookie` 를 탈취 | `HttpOnly` 속성 → JS에서 쿠키 접근 불가 |
| 스니핑 (패킷 도청) | HTTP 통신 중 쿠키가 탈취됨 | `Secure` 속성 → HTTPS에서만 전송 |
| CSRF (크로스 사이트 요청 위조) | 사용자 모르게 악성 사이트에서 요청 전송 | `SameSite=Strict` 또는 `Lax` (크롬 기본값) |

[ SameSite 설정 비교 ]

| 요청 유형 | Strict | Lax | None |
|---|:---:|:---:|:---:|
| 같은사이트요청 | O | O | O |
| GET 요청(외부링크 클릭) | X | O | O |
| POST 요청(외부 사이트) | X | X | O |
| iframe(외부이미지) | X | X | O |

```
// ── 보안 쿠키 설정 예시 (서버에서 Set-Cookie 헤더로 설정)
// JS에서 HttpOnly, Secure는 설정 불가 — 반드시 서버에서만 설정 가능

// Set-Cookie: sessionId=abc123; HttpOnly; Secure; SameSite=Strict; Path=/

// ── JS에서 설정 가능한 보안 옵션
document.cookie = "key=value; Secure; SameSite=Strict; path=/";

// ── encodeURIComponent 사용 — 특수문자·한글 포함된 값 안전하게 저장
const setCookieSafe = (key, value, day) => {
    const date = new Date();
    date.setDate(date.getDate() + day);
    document.cookie = [
        `${encodeURIComponent(key)}=${encodeURIComponent(value)}`,
        `expires=${date.toUTCString()}`,
        'path=/',
        'SameSite=Lax'
    ].join('; ');
};

// 읽을 때는 반드시 decodeURIComponent로 디코딩
const getCookieSafe = key => {
    const encoded = encodeURIComponent(key);
    const found = document.cookie.split('; ').find(c => c.startsWith(encoded + '='));
    return found ? decodeURIComponent(found.split('=')[1]) : null;
};
```

#### 웹 스토리지 (Web Storage)
웹 스토리지(Web Storage)란?
- 브라우저에 데이터를 key-value 형태로 저장하는 API.
- 쿠키와 달리 서버로 자동 전송되지 않으며, 용량이 훨씬 크고 JS로만 접근 가능.  

종류
- localStorage: 브라우저를 닫아도 데이터가 유지됨 (영구 저장).
- sessionStorage: 브라우저 탭(세션)이 닫히면 데이터가 삭제됨 (임시 저장).

[ localStorage vs sessionStorage ]

### 6. localStorage vs sessionStorage 세부 비교
| 구분 | localStorage | sessionStorage |
|---|---|---|
| 데이터 유지 기간 | 직접 삭제 전까지 영구 유지 | 탭(세션) 종료 시 자동 삭제 |
| 탭 간 공유 | 같은 도메인의 모든 탭에서 공유 | 탭마다 독립적 (공유 안 됨) |
| 브라우저 재시작 후 | 데이터 유지 | 데이터 삭제 |
| 서버 전송 | 자동 전송 안 됨 | 자동 전송 안 됨 |
| 용량 | 약 5~10MB | 약 5~10MB |
| 주요 용도 | 다크모드 설정, 자동로그인 여부, 즐겨찾기 | 입력 폼 임시 저장, 페이지 이동 간 데이터 유지 |

#### 웹 스토리지 API
- localStorage와 sessionStorage는 완전히 동일한 API를 사용.
- 객체 이름만 바꾸면 동일하게 동작.

[ 웹 스토리지 주요 메서드 ]

| 메서드 / 프로퍼티 | 설명 | 예시 |
|---|---|---|
| `setItem(key, value)` | 데이터 저장. `value`는 **반드시 문자열**. 객체는 `JSON.stringify` 필요. | `localStorage.setItem('name', '홍길동')` |
| `getItem(key)` | 데이터 읽기. 없으면 `null` 반환. | `localStorage.getItem('name')` |
| `removeItem(key)` | 특정 key 삭제. | `localStorage.removeItem('name')` |
| `clear()` | 해당 스토리지의 모든 데이터 삭제. | `localStorage.clear()` |
| `key(index)` | 인덱스로 key 이름 반환. | `localStorage.key(0)` |
| `length` | 저장된 항목 수. | `localStorage.length` |

```
// ── 문자열 저장 / 읽기 / 삭제
localStorage.setItem('name', '홍길동');
localStorage.getItem('name');    // '홍길동'
localStorage.removeItem('name');
localStorage.clear();            // 전체 삭제

// ── 객체 저장 — 반드시 JSON.stringify로 문자열 변환
const user = { id: 'ksmart', name: '홍길동', age: 25 };
localStorage.setItem('user', JSON.stringify(user));

// ── 객체 읽기 — JSON.parse로 다시 객체로 변환
const saved = JSON.parse(localStorage.getItem('user'));
console.log(saved.name); // '홍길동'

// ── null 체크 — 없는 key 읽으면 null 반환
const val = localStorage.getItem('없는키'); // null
if (val === null) console.log('데이터 없음');

// ── sessionStorage는 동일한 API 사용
sessionStorage.setItem('tempData', '임시값');
sessionStorage.getItem('tempData');
```

#### sessionStorage 실습 — 폼 임시 저장
- sessionStorage의 대표적인 활용 사례: 입력 중이던 폼 데이터를 임시 저장.
- 탭을 닫으면 삭제되고, 실수로 페이지를 새로고침해도 데이터가 복원됨.
- 아래 입력 후 새로고침해도 내용이 유지되는 것을 확인.

```
const $ssSaveBtn = document.getElementById('ssSaveBtn');
$ssSaveBtn.addEventListener('click', e => {
    const $input = document.querySelectorAll('.ssForm input');

    const valid = [...$input].some($input => {
        if(!$input.value){
            $input.focus();
            return true;
        }
    });
    if(valid) return;

    const tempForm = [...$input].reduce((acc, cur) => {
        const id = cur.id;
        const value = cur.value;
        acc[id] = value;
        return acc;

    sessionStorage.setItem('tempForm', JSON.stringify(tempForm));
});

const $ssClearBtn = document.getElementById('ssClearBtn');

const tempForm = sessionStorage.getItem('tempForm');

}
```

#### 객체 저장 — JSON.stringify / JSON.parse
- 웹 스토리지는 문자열(string)만 저장 가능.
- 배열·객체를 저장하려면 JSON.stringify로 문자열로 변환해서 저장하고,
- 꺼낼 때는 JSON.parse로 다시 원래 타입으로 변환해야 함.

[ JSON 변환 메서드 ]

| 메서드 | 방향 | 설명 |
|---|:---:|---|
| `JSON.stringify(value)` | 객체·배열 → 문자열 | 저장 전 반드시 사용. `undefined`, 함수, `Symbol` 은 변환 안 됨. |
| `JSON.parse(string)` | 문자열 → 객체·배열 | 읽은 후 반드시 사용. 잘못된 JSON 문자열이면 에러 발생 → `try-catch` 권장. |

```
const $objSaveBtn = document.getElementById('objSaveBtn');

$objSaveBtn.addEventListener('click', e => {

    $objResult.textContent = `id: ${savedInfo.objId}, 이름: ${savedInfo.objName}, 역할: ${savedInfo.objRole}`;

    $objResult.textContent = '삭제완료';
});
```

| **localStorage** | - 용량 큼 (5~10MB)<br>- 영구 저장 (탭 닫아도 유지)<br>- API 사용이 쿠키보다 단순 | - 서버로 전송 안 됨<br>- XSS 공격에 노출 가능<br>- 문자열만 저장 가능 (JSON 변환 필요) | - 다크모드·언어 설정<br>- 자동 로그인 여부<br>- 즐겨찾기·사용자 설정 |
