# 2장 React-Router 설정 및 예제 코드 요약

## 1. 개발 목표의 이해
- **SPA(Single Page Application)**: 리액트는 기본적으로 SPA로 동작하며, 여러 컴포넌트를 하나의 페이지에 표시합니다.
- **목적**: React-Router를 적용하여 브라우저 주소창(URL)에 따라 다른 화면(컴포넌트)이 표시되도록 구현합니다.
- **필요 모듈 설치**: 
  ```bash
  npm install react-router-dom
  ```

## 2. React-Router 설정
- `createBrowserRouter`를 통해 기본 라우터를 설정하고 파일로 작성합니다.
- SPA의 단점인 초기 로딩 속도를 높이기 위해 컴포넌트 렌더링을 지연시키는 `<Suspense>`와 `lazy()`를 적용합니다.

**`router/root.jsx` 예시 (기본 및 지연 로딩 설정)**
```jsx
import { Suspense, lazy } from "react";
import { createBrowserRouter } from "react-router-dom";

const Loading = <div>Loading....</div>
const Main = lazy(() => import("../pages/MainPage"))
const About = lazy(() => import("../pages/AboutPage"))

const root = createBrowserRouter([
  {
    path: "",
    element: <Suspense fallback={Loading}><Main/></Suspense>
  },
  {
    path: "about",
    element: <Suspense fallback={Loading}><About/></Suspense>
  }
])

export default root;
```

**`main.jsx` 예시 (제작한 라우터 연동)**
```jsx
import { createRoot } from 'react-dom/client'
import './index.css'
import { RouterProvider } from 'react-router-dom'
import root from './router/root'

createRoot(document.getElementById('root')).render(
  <RouterProvider router={root}></RouterProvider>
)
```

## 3. `<Link>`를 통한 이동
- SPA 방식이므로 브라우저 초기화를 방지하기 위해 `<a>` 태그 대신 `<Link>` 컴포넌트를 사용하여 이동합니다.

**`MainPage.jsx` 예시 (`<Link>` 사용법)**
```jsx
import { Link } from "react-router-dom";

const MainPage = () => {
  return (
    <div>
      <div className="flex">
        <Link to='/about'>About</Link>
      </div>
      <div className="text-3xl">Main Page</div>
    </div>
  );
}
export default MainPage;
```

## 4. 레이아웃 컴포넌트와 `children`
- 애플리케이션의 공통적인 틀(헤더, 사이드바 등)을 유지하도록 레이아웃을 생성합니다. 동적 콘텐츠 영역에는 `children`을 사용합니다.

**`BasicLayout.jsx` 예시**
```jsx
const BasicLayout = ({children}) => {
  return ( 
    <>
      <header className="bg-teal-400 p-5">   
        <h1 className="text-2xl md:text-4xl">Header</h1>
      </header>
      <div className="bg-white my-5 w-full flex flex-col space-y-4 md:flex-row md:space-x-4 md:space-y-0">   
        <main className="bg-sky-300 md:w-2/3 lg:w-3/4 px-5 py-40">
          {children} {/* 하위 컴포넌트들이 표시될 자리 */}
        </main>   
        <aside className="bg-green-300 md:w-1/3 lg:w-1/4 px-5 py-40">    
          <h1 className="text-2xl md:text-4xl"> Sidebar </h1>
        </aside>
      </div>
    </>
  );
}
export default BasicLayout;
```

페이지에 적용 시:
```jsx
import BasicLayout from "../layouts/BasicLayout";

const MainPage = () => {
  return (
    <BasicLayout>
      <div className="text-3xl">Main Page</div>
    </BasicLayout>
  );
}
export default MainPage;
```

## 5. 하위 경로 설정과 `<Outlet>`
- 특정 모듈(ex. Todo)에서 하위 경로를 추가하고 화면에 출력하려면 `<Outlet />`을 사용합니다.

**`todo/IndexPage.jsx` 예시**
```jsx
import { Outlet } from "react-router-dom";
import BasicLayout from "../../layouts/BasicLayout";

const IndexPage = () => {
  return (
    <BasicLayout>
      <div className="w-full flex m-2 p-2">
        <div className="text-xl m-1 p-2 w-20 font-extrabold text-center underline">LIST</div>
        <div className="text-xl m-1 p-2 w-20 font-extrabold text-center underline">ADD</div>
      </div>
      <div className="flex flex-wrap w-full">
        {/* 하위 컴포넌트가 마운트 될 영역 */}
        <Outlet/>
      </div>
    </BasicLayout>
  );
}
export default IndexPage;
```

## 6. 중첩 라우팅의 분리와 리다이렉션(Redirection)
- 경로가 늘어나면 편의상 외부로 라우팅 로직을 빼내어 관리하며, 초기 접속 시 특정 경로로 이동하려면 `<Navigate>`를 씁니다.

**`todoRouter.jsx` 예시 (`children`에 들어갈 배열을 리턴)**
```jsx
import { Suspense, lazy } from "react";
import { Navigate } from "react-router-dom";

const Loading = <div>Loading....</div>
const TodoList = lazy(() => import("../pages/todo/ListPage"))

const todoRouter = () => {
  return [
    {
      path: "list",
      element: <Suspense fallback={Loading}><TodoList/></Suspense>
    },
    { /* 만약 경로가 /todo 만 주어졌을 경우 /todo/list로 리다이렉션 함 */
      path: "", element: <Navigate replace to="list"/>
    }
  ]
}
export default todoRouter;
```

## 7. URL 정보 추출 기능 (`useParams` 및 `useSearchParams`)
- URL 상의 경로 변수는 `useParams()`로 읽어오며, 쿼리 스트링 파라미터는 `useSearchParams()`로 추출합니다.

**`ReadPage.jsx` 예시 (`useParams`)**
```jsx
import { useParams } from "react-router-dom";

const ReadPage = () => {
  const {tno} = useParams() // 경로가 "read/:tno" 일때 번호 추출
  return (
    <div className="text-3xl font-extrabold">
      Todo Read Page Component {tno}
    </div>
  );
};
export default ReadPage;
```

**`ListPage.jsx` 예시 (`useSearchParams`)**
```jsx
import { useSearchParams } from "react-router-dom";

const ListPage = () => {
  const [queryParams] = useSearchParams()
  const page = queryParams.get("page") || 1 // ?page=1
  const size = queryParams.get("size") || 10 // &size=10
  return ( 
    <div className="p-4 w-full bg-orange-200">
      <div className="text-3xl font-extrabold">Todo List Page Component</div>
      <div>Page: {page}</div>
      <div>Size: {size}</div>
    </div>
  );
}
export default ListPage;
```

## 8. 동적 페이지 이동 (`useNavigate`)
- 이벤트(버튼 클릭 등)를 통한 프로그래밍 방식의 이동을 처리할 때 사용합니다. `createSearchParams`를 곁들이면 상태(Query)를 유지한 채 이동하기 쉽습니다.

**`useNavigate` 활용 예시 (Read -> Modify 시 쿼리스트링 유지)**
```jsx
import { useCallback } from "react";
import { createSearchParams, useNavigate, useParams, useSearchParams } from "react-router-dom";

const ReadPage = () => {
  const {tno} = useParams()
  const navigate = useNavigate()
  
  // 현재 페이지와 사이즈의 쿼리스트링을 구함
  const [queryParams] = useSearchParams()
  const page = queryParams.get("page") ? parseInt(queryParams.get("page")) : 1
  const size = queryParams.get("size") ? parseInt(queryParams.get("size")) : 10
  
  // 생성 함수를 통해 다시 query 문자열을 생성 (`page=1&size=10`)
  const queryStr = createSearchParams({page, size}).toString()
  
  const moveToModify = useCallback((tno) => {
    // 이동 시 경로에 검색 조건(search)도 같이 붙여서 이동 처리
    navigate({ pathname: `/todo/modify/${tno}`, search: queryStr })
  }, [tno, page, size])

  return ( 
    <div className="text-3xl font-extrabold">
      Todo Read Page Component {tno}
      <div> 
        <button onClick={() => moveToModify(tno)}>Test Modify</button> 
      </div>
    </div>
  );
}
export default ReadPage;
```
