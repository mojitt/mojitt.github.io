## JavaScript 브라우저 객체 (BOM: Browser Object Model)
BOM이란?
- 브라우저와 관련된 정보 및 기능을 제공하는 객체들의 집합.
- 최상위 객체는 window이며, 모든 전역 변수·함수·객체는 window의 프로퍼티/메서드.
- window 식별자는 생략 가능 (예: window.alert() → alert())

[ window 하위 객체 구성 ]

| 하위 객체 | 설명 |
| :--- | :--- |
| `document` | HTML 문서 정보 및 DOM 조작 |
| `location` | 현재 페이지의 URL(주소창) 정보 및 이동 |
| `history` | 브라우저 방문 기록 및 페이지 이동 |
| `navigator` | 사용자의 브라우저 정보 및 운영체제 정보 |
| `screen` | 사용자 디바이스(화면) 크기 정보 |

#### window 메서드
- 브라우저 창을 제어하거나 사용자와 상호작용하는 메서드를 제공.
- window는 생략 가능하므로 window.alert()와 alert()는 동일.

**open() / close()**
- 새 창(팝업)을 열거나 닫을 때 사용.
- open(url, target, options) 형식으로 사용.

[ window.open() 파라미터 ]

| 파라미터 | 설명 | 예시 |
| :---: | :--- | :--- |
| `url` | 열 페이지의 주소 | `'https://example.com'` |
| `target` | 창 생성 방식 | `'_blank'` (새 창), `'_self'` (현재 탭) |
| `options` | 새 창의 크기·위치 옵션 (문자열) | `'width=400,height=300,left=100,top=100'` |

[ open() options 목록 ]

| 옵션 | 설명 | 예시 |

| `left` | 새 창의 왼쪽 위치 (px) | `left=100` |
| `top` | 새 창의 위쪽 위치 (px) | `top=100` |

#### location 객체
- 현재 URL(주소창)의 정보를 가지고 있는 객체.
- 프로토콜, 호스트, 포트, 경로 등의 정보를 제공하며, 페이지 이동도 가능.

[ location 주요 프로퍼티 ]

| 프로퍼티 | 설명 | 예시 값 |
| :---: | :--- | :--- |
| `location.href` | 현재 페이지의 전체 URL (값 대입 시 해당 주소로 이동) | `'https://example.com:8080/path?q=1#top'` |
| `location.protocol` | 프로토콜 (콜론 포함) | `'https:'` |
| `location.host` | 호스트명 + 포트 | `'example.com:8080'` |
| `location.hostname` | 호스트명 (포트 제외) | `'example.com'` |
| `location.port` | 포트 번호 | `'8080'` |
| `location.pathname` | 경로 (도메인 이후 부분) | `'/path'` |
| `location.search` | 쿼리스트링 (? 포함) | `'?q=1'` |
| `location.hash` | 해시 (# 포함) | `'#top'` |

[ location 주요 메서드 ]

| 메서드 | 설명 | 특징 |
| :---: | :--- | :---: |
| `location.assign(url)` | 해당 url로 이동 (방문 기록 O) | 뒤로가기 가능 |
| `location.replace(url)` | 해당 url로 이동 (방문 기록 X) | 뒤로가기 불가 |
| `location.reload()` | 현재 페이지 새로고침 | F5와 동일 |

```
console.log(location);
console.log(location.href);
console.log(location.host);
//location.href = 'https://ksmart.or.kr'; // 방문기록 남음
//location.assign('https://ksmart.or.kr'); // 방문기록 남음
//location.replace('https://ksmart.or.kr'); // 방문기록 기록되지 않음
//location.reload(); // 새로고침
```

- 브라우저의 방문 기록 정보를 가진 객체.
- 앞으로 가기, 뒤로 가기 등 페이지 이동을 제어할 수 있음.

[ history 주요 프로퍼티 / 메서드 ]

| :---: | :--- | :--- |

## JavaScript 문서객체 (DOM: Document Object Model)
DOM이란?
- 브라우저가 HTML 문서를 파싱하여 생성하는 트리 자료구조.
- HTML 문서의 계층적 구조와 정보를 표현하며, 노드 객체들로 구성됨.
- DOM을 통해 HTML의 구조·내용·스타일을 동적으로 조작 가능.

#### DOM 노드 타입

[ DOM 주요 노드 타입 ]

| 노드 타입 | 설명 | 예시 |
| :---: | :--- | :---: |
| 문서 노드 | DOM 트리 최상위 루트 노드. `document` 객체를 가리킴. 모든 노드 접근의 진입점. | `document` |
| 요소 노드 | HTML 요소를 가리키는 객체. 부모-자식 관계를 가지며 문서의 구조를 표현. | `<div>`, `<ul>` |
| 어트리뷰트 노드 | HTML 요소의 속성을 가리키는 객체. 부모 노드와 연결되지 않으므로 요소 노드를 먼저 취득해야 접근 가능. | `class="ksmart"` |
| 텍스트 노드 | HTML 요소의 텍스트를 가리키는 객체. DOM 트리의 최하위(리프 노드). 문서의 정보를 표현. | `ks58` |

```

<div id="ksmart58gi">ks58</div>
```

## 요소 노드 취득
- HTML의 구조·내용·스타일을 동적으로 조작하려면 먼저 요소 노드를 취득해야 함.
- 취득 방법: getElementById, getElementsByTagName, getElementsByClassName, querySelector, querySelectorAll

#### getElementById()
- document.getElementById(id)
- 인수로 전달한 id 속성값을 가진 요소 노드를 반환. 존재하지 않으면 null 반환.
- 반드시 문서 노드(document)를 통해 호출.
- 내부적으로 id 인덱스를 사용하므로 querySelector보다 빠름.

```
const $ks301 = document.getElementById('ks301');
console.log({ $ks301 });
console.log($ks301.id); // ks301
const $ks303 = document.getElementById('ks303');
//console.log($ks303.id); // null.id, 에러 시 다음 코드 실행 불가
console.log('뒤에 프로세스');
```

#### getElementsByTagName()
- document.getElementsByTagName(tagName)
- 인수로 전달한 태그 이름을 가진 모든 요소 노드를 HTMLCollection으로 반환.
- 모든 요소를 취득할 때는 '*'를 인수로 전달. 존재하지 않으면 빈 HTMLCollection 반환.
- HTMLCollection은 live 객체이므로 순회 중 DOM이 변경되면 예상치 못한 동작 발생 주의.

```
const $allTag = document.getElementsByTagName('*');
console.log($allTag);
const $divTags = document.getElementsByTagName('div');
console.log($divTags);
[...$divTags].forEach(div => console.log(div));
/*
for(const element of $divTags){
    console.log(element);
}
*/
```

#### getElementsByClassName()
- document.getElementsByClassName(className)
- 인수로 전달한 class 속성값을 가진 모든 요소 노드를 HTMLCollection으로 반환.
- HTMLCollection은 live 객체이므로 순회 중 변경 시 주의 필요 → 스프레드로 배열 변환 권장.

#### querySelector() / querySelectorAll()
- document.querySelector(cssSelector) : CSS 선택자를 만족하는 첫 번째 요소 반환. 없으면 null.
- document.querySelectorAll(cssSelector) : CSS 선택자를 만족하는 모든 요소를 NodeList로 반환. 없으면 빈 NodeList.
- 반환된 NodeList는 정적(static) 객체 → DOM이 변경되어도 반영되지 않음. getElementsBy~ 계열과 다름.
- CSS 선택자 문법이 맞지 않으면 DOMException 에러 발생.

[ CSS 선택자 종류 ]

| 선택자 | 설명 | 예시 |
| :---: | :--- | :---: |
| `*` | 모든 요소 선택 | `*` |
| 태그 | 태그 이름으로 선택 | `input` |
| `#id` | id 속성으로 선택 | `#exLabel` |
| `.class` | class 속성으로 선택 | `.exLabel` |
| `태그[속성=값]` | 속성값이 일치하는 요소 선택 | `input[type="text"]` |
| `A B` | A의 자손 중 B 선택 (자손 선택자) | `div label` |
| `A > B` | A의 직계 자식 중 B 선택 (자식 선택자) | `ul > li` |
| `A + B` | A 바로 다음 형제 B 하나 선택 (인접 형제) | `div + label` |
| `A ~ B` | A 다음 모든 형제 B 선택 (일반 형제) | `div ~ label` |

```
const $ksmart301 = document.querySelector('.ksmart301');
console.log($ksmart301);
const $ksmart303 = document.querySelector('.ksmart303');
console.log($ksmart303); // 없을 경우 null 반환

// 취득 시 exception 주의 (css 선택자 문법에 맞게 작성해야한다.)
//const $ksmartCourse = document.querySelector('div.example < ul');

const $ksmartCourse = document.querySelectorAll('.ksmart-course');
console.log($ksmartCourse);
const $ksmartCourse2 = document.querySelectorAll('.ksmart-course2');
console.log($ksmartCourse2); // 없을 경우 빈 NodeList 반환
//$ksmartCourse.forEach(element => console.log(element));
```

#### matches()
- element.matches(cssSelector)
- 취득한 요소가 인수로 전달된 CSS 선택자로도 취득 가능한지 확인. true / false 반환.
- 이벤트 위임에서 특정 요소를 판별할 때 유용하게 사용.

```
const $ksmart58 = document.querySelector('.ksmart58');
console.log($ksmart58.matches('li.ksmart58'));
console.log($ksmart58.matches('#ksmart > li.ksmart58'));
console.log($ksmart58.matches('div.example li.ksmart58'));
console.log($ksmart58.matches('ul.ksmart58')); // false
```

#### HTMLCollection vs NodeList

[ HTMLCollection vs NodeList 비교 ]

| 구분 | HTMLCollection | NodeList |
| :---: | :---: | :---: |
| 반환 메서드 | `getElementsByTagName`, `getElementsByClassName` | `querySelectorAll`, `childNodes` |
| live 여부 | live (DOM 변경 즉시 반영) | 대부분 static (`querySelectorAll`), `childNodes`는 live |
| forEach 사용 | 불가 (스프레드/`Array.from` 변환 필요) | 가능 |
| 순회 권장 방법 | `[...collection].forEach()` | `nodeList.forEach()` |


## DOM 조작 (DOM Manipulation)
DOM 조작이란?
- 새로운 노드를 생성하여 DOM에 추가하거나, 기존 노드를 삭제·교체·복사하는 것.
- 주요 방법: innerHTML, textContent, nodeValue, createElement / append / remove 등

- 요소 노드를 취득한 다음, 해당 노드를 기점으로 부모·형제·자식 노드를 탐색.
- 텍스트 노드 주의: HTML 태그 사이의 줄바꿈·공백도 텍스트 노드로 생성됨.
- 요소 노드만 탐색하려면 Element가 붙은 프로퍼티 사용 권장.

#### 자식 노드 탐색

[ 자식 노드 탐색 프로퍼티 ]

| 프로퍼티 | 설명 | 반환 타입 |
| :---: | :--- | :---: |
| `childNodes` | 자식 노드 모두 반환 (텍스트 노드 포함) | `NodeList` |
| `children` | 자식 중 요소 노드만 반환 | `HTMLCollection` |
| `firstChild` | 첫 번째 자식 노드 (텍스트 노드 포함) | `Node` |
| `lastChild` | 마지막 자식 노드 (텍스트 노드 포함) | `Node` |
| `firstElementChild` | 첫 번째 자식 요소 노드만 반환 | `Element` |
| `lastElementChild` | 마지막 자식 요소 노드만 반환 | `Element` |
| `hasChildNodes()` | 자식 노드 존재 여부 확인 | `boolean` |

#### 부모 / 형제 노드 탐색

[ 부모 · 형제 노드 탐색 프로퍼티 ]

| 프로퍼티 | 설명 |
| :---: | :--- |
| `parentNode` | 부모 노드 반환 (텍스트 노드는 부모 노드가 될 수 없음) |
| `closest(selector)` | 자신을 포함한 조상 중 CSS 선택자와 일치하는 가장 가까운 노드 반환. 없으면 null |
| `previousSibling` | 이전 형제 노드 반환 (텍스트 노드 포함) |
| `nextSibling` | 다음 형제 노드 반환 (텍스트 노드 포함) |
| `previousElementSibling` | 이전 형제 요소 노드만 반환. 없으면 null |
| `nextElementSibling` | 다음 형제 요소 노드만 반환. 없으면 null |

```
const $ksmartRoom = document.getElementById('ksmartRoom');
console.log($ksmartRoom);

// 자식노드탐색
console.log($ksmartRoom.childNodes); // 요소노드, 텍스트노드 9개
console.log($ksmartRoom.childNodes[1]);
console.log($ksmartRoom.children); // 요소노드만
console.log($ksmartRoom.children[0]);

// 첫번째/마지막 자식노드 탐색
console.log($ksmartRoom.firstChild); // #text
console.log($ksmartRoom.firstElementChild); // li.room301
console.log($ksmartRoom.lastChild); // #text
console.log($ksmartRoom.lastElementChild); // li.counseling-room

// 형제노드(이전/이후) 탐색
const $room301 = $ksmartRoom.children[0];
console.log($room301);
console.log($room301.previousSibling); // #text, 텍스트노드 포함
// 이전 형제노드가 없을 시 null 반환
console.log($room301.previousElementSibling); // null
console.log($room301.nextSibling); // #text, 텍스트노드 포함
console.log($room301.nextElementSibling); // li.room302

// 자식노드 확인
console.log($ksmartRoom.hasChildNodes()); // true, 텍스트노드 포함한 자식노드 존재여부
console.log($ksmartRoom.children.length > 0); // 자식 요소 노드 존재여부 확인
```

#### 콘텐츠 조작

[ 콘텐츠 조작 프로퍼티 비교 ]

| 프로퍼티 | 대상 | HTML 마크업 | 특징 |
| :---: | :---: | :---: | :--- |
| `innerHTML` | 요소 노드 | 파싱·반영 | HTML 문자열로 자식 노드 전체 교체. XSS 취약점 주의. |
| `textContent` | 요소 노드 | 무시 (텍스트로 처리) | 자손 노드의 텍스트 전체 반환. HTML 태그 삽입 불가. |
| `nodeValue` | 텍스트 노드 | 해당 없음 | 텍스트 노드에서만 문자열 반환. 요소 노드에서는 null. |

- XSS(Cross-Site Scripting) 주의: 공격자가 악성 스크립트를 페이지에 삽입하여 다른 사용자의 브라우저에서 실행되도록 하는 공격.
- innerHTML은 문자열을 HTML로 파싱하기 때문에 사용자 입력값을 직접 삽입하면 스크립트가 실행될 수 있음.
- 사용자 입력값은 반드시 textContent를 사용하거나 이스케이프 처리 후 삽입할 것.

```
const $ksmart = document.getElementById('ksmart');
console.log($ksmart.innerHTML); // 취득 요소의 자식노드 문자열 반환
console.log($ksmart.outerHTML); // 취득 요소 태그 포함한 문자열 반환
//$ksmart.innerHTML = '
한국스마트정보교육원 개발자 58기
';
console.log($ksmart.innerText); // 텍스트노드 문자열 반환
console.log($ksmart.outerText); // 텍스트노드 문자열 반환
$ksmart.innerText = '
한국스마트정보교육원 개발자 58기
';

const facilties = ['강의실301호', '강의실302호', '교직원실', '행정실', '상담실'];
const $exDiv = document.getElementById('exDiv');
const liStr = facilties.map(place => `
${place}
`).join('');
$exDiv.innerHTML = `
${liStr}
`;

console.log('====XSS 실습====');
const $xssInput = document.getElementById('xssInput');
const $xssDangerBtn = document.getElementById('xssDangerBtn');
const $xssSafeBtn = document.getElementById('xssSafeBtn');
const $xssResult = document.getElementById('xssResult');

$xssDangerBtn.addEventListener('click', function(){
    $xssResult.innerHTML = $xssInput.value;
});
$xssSafeBtn.addEventListener('click', function(){
    $xssResult.textContent = $xssInput.value;
});
```

#### 노드 생성과 추가

[ 노드 생성 메서드 ]

| 메서드 | 설명 | 예시 |
| :---: | :--- | :--- |
| `document.createElement(tag)` | 요소 노드 생성 | `document.createElement('li')` |
| `document.createTextNode(text)` | 텍스트 노드 생성 | `document.createTextNode('안녕')` |
| `document.createDocumentFragment()` | DocumentFragment 생성 (서브 DOM 구성용 컨테이너) | `document.createDocumentFragment()` |

[ 노드 삽입 메서드 ]

| 메서드 | 설명 | 원본 변경 |
| :---: | :--- | :---: |
| `append(노드, ...)` | 자식 노드 끝에 삽입 (여러 노드·문자열 가능) | O |
| `prepend(노드, ...)` | 자식 노드 앞에 삽입 | O |
| `before(노드, ...)` | 해당 노드 이전(형제)에 삽입 | O |
| `after(노드, ...)` | 해당 노드 이후(형제)에 삽입 | O |
| `appendChild(노드)` | 자식 노드 끝에 삽입 (노드 하나만 가능) | O |
| `insertBefore(새노드, 위치노드)` | 위치노드 앞에 새노드 삽입. 위치노드가 null이면 끝에 삽입. | O |

[ 노드 교체 · 삭제 · 복사 메서드 ]

| 메서드 | 설명 |
| :---: | :--- |
| `replaceChild(새노드, 대상노드)` | 대상 노드를 새 노드로 교체 |
| `replaceChildren(...노드)` | 모든 자식 노드를 인수로 교체. 인수 없으면 자식 노드 전체 삭제. |
| `removeChild(노드)` | 인수로 전달한 자식 노드 삭제 |
| `remove()` | 취득한 요소 노드 자신을 DOM에서 제거 |
| `cloneNode(deep)` | 노드 복사. true: 자손 전체 복사, false(기본값): 노드 자신만 복사. 이벤트는 복사 안 됨. |
