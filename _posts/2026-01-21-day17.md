#### 인스턴스 멤버
객체(인스턴스) 마다 가지고 있는 필드와 메소드  
인스턴스 멤버는 객체 소속된 멤버이기 때문에 객체가 없이 사용불가

```
//Person2.class

String name;
	
void hello() {
	System.out.println("안녕~~!");
}
```

```
//Main.class

Person2 p1 = new Person2();

	p1.name = "홍길동"; // 객체의 name 필드이므로 인스턴스 필드
		
	/*
	 인스턴스 필드란,
	 객체마다 다른 값을 저장할 수 있는 객체의 필드를 의미합니다.
	 사용하기 위하여 반드시 객체부터 생성해야 합니다. 
	 */
		
	p1.hello(); // 객체의 기능인 hello 메소드 이므로 인스턴스 메소드
		
	/*
	 인스턴스 메소드랑,
	 객체를 통하여 객체마다 호출할 수 있는 객체의 기능(메소드)를 의미합니다.
	 사용하기 위하여 반드시 객체부터 생성해야 합니다.		  
	 */
		
	/*
	 인스턴스 필드, 인스턴스 메소드 -> 인스턴스 멤버
	 */
```

#### this
객체(인스턴스) 자신의 참조(번지)를 가지고 있는 키워드  
객체 내부에서 인스턴스 멤버임을 명확히 하기 위해 this. 사용  
매개변수와 필드명이 동일할 때 인스턴스 필드임을 명확히 하기 위해 사용

```
//Person2.class

String name;

void info(String name) {	
	System.out.println("현재 객체의 이름 : "+name); // 매개값을 출력 -> "고길동"
	System.out.println("현재 객체의 이름 : "+this.name); // 필드값을 출력 -> "홍길동"
}
```

```
//Main.class

Person2 p1 = new Person2();
p1.name = "홍길동";

p1.info("고길동");

/*
 this 키워드는 필드 또는 메소드를 호출하고 있는
 객체의 주소값을 가리키는 키워드 입니다.
 보통 생략하여 클래스 내부에서 사용가능 합니다.
 하지만 매개변수와 필드명이 중복되는 경우
 this 키워드를 사용하여 필드를 구분합니다.
 */
```

## 정적 멤버와 static
#### 정적 멤버란
클래스에 고정된 필드와 메소드 - 정적 필드, 정적 메소드  
정적 멤버는 클래스에 소속된 멤버  
- 객체 내부에 존재하지 않고, 메소드 영역에 존재
- 정적 멤버는 객체를 생성하지 않고 클래스로 바로 접근해 사용

선언) 필드 또는 메소드 선언할 때 static 키워드 붙임  

```
//Calculator.class

String color; // 인스턴스 멤버 (계산기마다 색상이 다르므로)
static double pi = 3.14; // 모든 계산기의 원주율은 늘 3.14 객체 생성없이 바로 사용(클래스로) 가능하도록

static String feild1;
static {
		
	/*
	 정적 블럭
	 - 정적 필드를 초기화하기 위하여 연산식 또는 기타 실행문을 작성할 수 있는 영역입니다.
	 - 클래스가 로드되는 시점 단 한번만 실행됩니다.
	 - 정적 멤버는 정적 블럭에서 사용할 수 있습니다.
	 - 인스턴스 멤버는 정적블럭셍서 사용할 수 없습니다.
	 - 혹시 굳이 인스턴스멤버가 필요한 경우 객체를 생성하여 사용할 수 있습니다.
	 */
	
	if(true) {
		System.out.println("정적 블록");
	}
	
	feild1 = "정적"+"필드";
	int result = plus(3, 2);
	
	// color = "파랑"; 인스턴스 필드 사용 X
	// info(); 인스턴스 메소드 사용 X
	Calculator c = new Calculator(); // 객체를 생성해주면 사용가능
	c.color = "파랑";
	c.info();
}
	
/*
 모든 객체의 공통값 -> 정적 필드
 메소드 내부에서 객체의 필드 또는 메소드를 사용하지 않는 경우 -> 정적 메소드
 */

static int plus(int num1, int num2) {
	// 객체의 필드 또는 메소드를 사용하지 않으므로 정적 메소드 사용
	return num1+num2;
}

void info() {
	// 인스턴스 필드인 color를 메소드 내부에서 사용하므로 인스턴스 메소드여야 합니다.
	System.out.println("계산기의 색상은 "+color+"색입니다.");
}
	
public static void main(String[] args) {

	System.out.println("원주율 : "+Calculator.pi); // 정적 필드인 pi는 객체 생성 없이 클래스 명으로 사용가능

	int result = Calculator.plus(3, 1);
	System.out.println("Calculator.plus(3, 1) -> "+result);

	//Calculator.info(); 정적 메소드가 아니므로 사용X
	
	Calculator calc = new Calculator();
	calc.color = "파랑";
	calc.info(); // 새 객체를 생성해줘야 사용 가능

	System.out.println(calc.pi); // 정적 멤버는 인스턴스로 호출하는 것을 권장하지 않습니다.
}
```

#### final
```
//Person2.class

String name;
final String jumin;
final String nation = "한국"; 
int age;
	
static final int MAX_AGE = -1; // 상수 선언 : static final 타입 상수명, 상수란 : 값이 변하지 않고 단 하나만 존재하는 절대적인 값
		
void setAge(int age) {
	//maxAge = -100; final 키워드 이기 때문에 변경x
	if(age > MAX_AGE) {
		this.age = age;	
	}		
}
	
//Person2(){} jumin 필드는 final 키워드를 가지고 있으므로 필드의 초기값을 대입하거나 생성자에게 반드시 초기화 해야합니다.
	
Person2(String jumin){
	this.jumin = jumin;
}

public static void main(String[] args) {
		
	Person2 p1 = new Person2("123456-1111111");
		
	p1.name = "홍길동";
	//p1.jumin = "123456-1111111"; final 필드는 객체 생성 후 값수정 X
	
	System.out.println("이름 : "+p1.name+", 주민번호 : "+p1.jumin+", 국적 : "+p1.nation);
	
	//p.nation = "미국"; final 필드는 값이 초기화 되면 변경할 수 없습니다.
		
	p1.setAge(-100);	
	System.out.println("나이: "+p1.age);
		
	System.out.println("byte 타입 최솟값 : "+Byte.MIN_VALUE);
}
```
