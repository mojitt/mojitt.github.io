## 클래스

## 필드
- 클래스로 생성된 각 객체가 갖는 고유 정보 데이터를 의미합니다.
- 파이썬에서는 객체 생성 이후 또는 메소드, 생성자를 통하여 필드를 선언할 수 있습니다.

#### 1. 객체 생성 이후 필드 초기화
- 파이썬에서 속성은 같은 클래스로 생성된 객체여도 다른 속성을 가질 수 있습니다.
- 속성 값 변경시 오타를 내면 새로운 속성으로 추가하여 오류x

```
class Person2:
    pass

p1 = Person2()
p2 = Person2()

p1.name = '사람1' # p1 객체에만 name 속성 추가

print(p1.name)
#print(p2.name) # p2 객체에는 name 속성이 없으므로 오류 발생
```

#### 2. 메소드, 생성자를 활용하여 필드 초기화
- 생성자를 사용하여 필요한 필드들을 미리 선언하여 초기화하면 모든 객체들이 해당 속성을 가지고 태어나게 됩니다.
- 객체가 생성된 후에 필드값은 변경 할 수 있습니다. (주의, 변경시 필드명 오타나면 새로운 필드 선언으로 간주됨)


## 메소드
- 객체들이 공통으로 갖는 기능을 의미합니다.
- 파이썬에서는 메소드에서도 필드를 선언 할 수 있습니다.
- 파이썬에서는 메소드 역시 함수로 간주합니다.

```
class Tester:
    def check(self): # 메소드 선언, 첫번째 매개변수로는 무조건 객체 자신 전달 받습니다.
        return '데이터 셋팅' # 메소드 호출한 위치로 반환 값 전달

    def setData(self, tester_name, task, confirm_yn):
        self.tester_name = tester_name # 필드 선언과 초기화
        self.task = task # 필드 선언과 초기화
        self.confirm_yn = confirm_yn # 필드 선언과 초기화

    def infoData(self):
        return self.tester_name, self.task, self.confirm_yn

t1 = Tester()
#result = t1.infoData() # setData 메소드를 호출한 후에만 infoData 메소드를 호출할 수 있습니다. (필드 초기화 때문)

result = t1.setData('홍길동', '회원가입', True) # return이 없으면 None 반환
print(result)

result = t1.infoData()
print(result)
```

#### 클래스 메소드, 인스턴스 메소드, 정적 메소드

```
class Sample:
    class_field = '클래스 변수1'
    
    def __init__(self):
        self.instance_field = '인스턴스 변수1'
    
    def instance_method(self): # 인스턴스 메소드
        return '인스턴스 메소드'
    
    @classmethod
    def class_method(cls):
        #return f'클래스 메소드, {cls.class_field},{cls.instance_field}' # 클래스 메소드에서 인스턴스 멤버 사용x
        return f'클래스 메소드, {cls.class_field}'
    
    @staticmethod # 정적메소드 선언으로 첫번째 인수(self,class)가 자동으로 전달되지 x
    def static_method(arg1,arg2):
        print(f'{arg1},{arg2}')
    
s1 = Sample()
print(s1.instance_method())
print(Sample.instance_method(s1)) # 인스턴스 메소드는 클래스를 통하여 호출할 수 있으나 첫번째 인수로 객체를 넘겨야 합니다.

print(Sample.class_method())

s1.static_method('a','b') # 객체를 통하여 정적메소드 호출
Sample.static_method('c','d') # 객체를 통하여 정적메소드 호출
```

## 상속
- 상위클래스가 갖는 구성요소를 상속받아 하위클래스에서 사용하는 것을 의미합니다.

```
class 자식클래스(부모클래스):
    def __init__(self):
        super().__init__() # 부모객체의 필드를 초기화하여 상속
        
    # 부모의 필드와 메소드를 상속됩니다.
```

```
class Parent:
    def __init__(self):
        self.p_field1 = '부모 필드1'
        
    def p_method(self):
        print('부모 메소드 호출')
        
        
class Child1(Parent): # Child1에 Parent 클래스 상속
    def __init__(self):
        super().__init__() # 부모 생성자 호출하여 부모 필드 초기화
        self.c1_field = '자식1 필드1'
    
    def c1_methodll(self):
        print('자식1 메소드 호출')
        
c1_1 = Child1()
print(c1_1.p_field1)
c1_1.p_method()


class Parent2:
    def setField(self):
        self.p2_field1 = '부모2의 필드1'

class Child2(Parent2):
    def __init__(self):
        super().setField() # 부모 클래스에서 setField() 메소드가 필드 초기화 역할이므로 super() 함수를 통하여 호출

c2 = Child2()
print(c2.p2_field1)

# 파이썬에서 부모객체의 필드는 초기화하는 메소드가 호출되어야 존재합니다.

class Parent3:
    def __init__(self):
        self.p3_field1 = '부모3 필드1'

class Child3(Parent3):
    def init_parent(self): #Child3이 생성될 때가 아닌 별도의 메소드로 부모필드 초기화
        super().__init__()
        
    def use_super_field(self): # 부모객체의 필드를 사용하는 메소드
        print('use_super_field() 호출')
        #print(super().p3_field1)
        print(self.p3_field1) # 부모 필드를 접근할 때는 self 객체를 사용합니다.

c3 = Child3()
c3.init_parent() # 이 순간에 부모 필드들 생성
print(c3.p3_field1)
c3.use_super_field()
```

#### 다중 상속
- 파이썬에서는 하나의 자식이 여러 부모를 상속받을 수 있습니다.
- MRO 구조 : 상속 구조에서의 객체들 체이닝 구조
- Chil4 -> Parnet -> Parent2 : 자신 클래스를 제외한 super() 함수가 메소드를 탐색하는 순서

```
class Child4(Parent, Parent2):
    def __init__(self):
        super().__init__() # Parent의 __init__() 호출
        super().setField() # Parent2의 setField() 호출
    
    def check_parent(self):
        self.p_method() # 자식 클래스 내부에서 부모의 메소드
        #self.setField() # 자식 클래스 내부에서 부모의 메소드

c4 = Child4()
c4.check_parent()
c4.p_method() # 자식 객체를 통하여 부모의 메소드 호출
#c4.setField() # 자식 객체를 통하여 부모의 메소드 호출

print(c4.p_field1)
print(c4.p2_field1)


class Test1:

```

#### 메소드 오버라이딩
부모가 갖는 메소드를 자식에서 재정의하는 것을 의미합니다.

```
class OverParent:
    def over_method(self):
        print('재정의 전 부모의 메소드입니다.')
    
class OverChild(OverParent):
    pass

over_child = OverChild()
over_child.over_method()

class OverChild2(OverParent):
    # 자식에서 부모 메소드 재정의
    # 선언부 확인하는 데코레이터는 없으므로 메소드명 오타 발생시 새로운 메소드 선언이 되므로 주의!
    def over_method(self):
        return '내가 재정의 했지롱!'
    
over_child2 = OverChild2()
print(over_child2.over_method())
```
