## JavaScript 예외처리 (Exception Handling)
예외처리란?
- 실수, 예상치 못한 사용자 입력, 잘못된 서버 응답 등으로 인해 스크립트 실행이 중단되는 것을 방지하기 위한 처리.
- 주요 구문: try, catch, finally, throw

#### try...catch 구문
- 에러가 발생해도 catch에서 처리하기 때문에 스크립트 실행이 중단되지 않음.
- 에러 객체(error)는 name, message, stack 프로퍼티를 포함.

[ try...catch 동작 순서 ]

| 단계 | 설명 |
|-----|-----|
| 1단계 | `try {}` 안의 코드 실행 |
| 2단계 (정상) | 에러가 없다면 try 마지막 줄까지 실행, catch 블록은 건너뜀 |
| 3단계 (에러) | 에러가 있다면 try 코드 실행이 중단되고 `catch(err)` 블록 실행 |

[ Error 객체 프로퍼티 ]

| 프로퍼티 | 설명 | 예시 |
|---------|------|------|
| `error.name` | 에러 이름 (에러 종류) | ReferenceError, TypeError, SyntaxError |
| `error.message` | 에러 상세 내용을 담고 있는 메시지 | `'age is not defined'` |
| `error.stack` | 에러가 발생한 호출 스택 정보 | `ReferenceError: age is not defined at ...` |

[ 주요 에러 종류 ]

| 에러 타입 | 설명 | 발생 예시 |
|----------|------|----------|
| SyntaxError | 문법 오류 | `console.log(가)` |
| ReferenceError | 존재하지 않는 변수 참조 | `console.log(undeclaredVar)` |
| TypeError | 타입이 잘못된 경우 | `null.property` |
| RangeError | 값이 허용 범위를 벗어난 경우 | `new Array(-1)` |
| URIError | 잘못된 URI 처리 | `decodeURIComponent('%')` |

```
try{
    console.log(name);
    let name;
}catch(error){
    console.log(error instanceof Error);
    console.log(error.name);
    console.log(error.message);
    console.log(error.stack);
}finally{
    console.log('그 다음코드');
}
```

#### try...catch...finally 구문
- finally 블록은 에러 발생 여부와 관계없이 항상 실행됨.
- 주로 리소스 해제, 로딩 상태 종료 등 반드시 실행해야 하는 코드에 사용.

[ try...catch...finally 동작 순서 ]

| 단계 | 정상 실행 | 에러 발생 |
|-----|-----------|-----------|
| 1단계 | try 블록 실행 | try 블록 실행 중단 |
| 2단계 | catch 블록 건너뜀 | catch 블록 실행 |
| 3단계 | finally 블록 실행 (항상) | finally 블록 실행 (항상) |

#### throw (예외 발생)
- 개발자가 의도적으로 에러를 발생시킬 때 사용.
- throw 뒤에는 어떤 값이든 올 수 있지만, 주로 Error 객체를 사용.

[ throw 사용법 ]

| 방법 | 예시 | 설명 |
|-----|-----|-----|
| 기본 에러 | `throw new Error('에러 메시지')` | 일반적인 에러 발생 |
| 타입 에러 | `throw new TypeError('타입 에러')` | 타입 관련 에러 발생 |
| 범위 에러 | `throw new RangeError('범위 에러')` | 값 범위 관련 에러 발생 |
| 문자열 | `throw '에러 발생'` | 문자열도 throw 가능 (비권장) |

```
const absIntFn = (data) => {
    if(typeof data !== 'number') throw new Error('입력하신 데이터는 숫자가 아닙니다.');
    if(!Number.isInteger(data)) throw new RangeError('입력하신 숫자는 정수가 아닙니다.');
    return Math.abs(data)
}

try{
    const result = absIntFn(123.123);
    console.log(result);
}catch(error){
    console.log(error);
    console.log(error.name);
    console.log(error.message);
    console.log(error.stack);            
}
```

#### 에러 타입 구분 (instanceof)
catch 블록에서 instanceof를 사용해 에러 종류를 구분하여 다르게 처리할 수 있음.

```
const data = ['40', 40, 40.123];

data.forEach(data => {
    try {
        absIntFn(data);
    } catch (error) {
        if(error instanceof RangeError){
            console.log('rangeError', error.message);
        }else if(error instanceof TypeError){
            console.log('typeError', error.message);
        }else{
            console.log('error', error.message);
        }
    }
});
```

#### 사용자 정의 예외 (Custom Error)
- Error 클래스를 상속(extends)하여 프로젝트에 맞는 커스텀 에러 클래스를 직접 만들 수 있음.
- instanceof로 에러 종류를 구분할 수 있고, name·message·stack 등 기본 프로퍼티도 그대로 사용 가능.
- 필요에 따라 추가 프로퍼티(예: statusCode, field 등)를 자유롭게 확장할 수 있음.

[ 사용자 정의 예외 클래스 구조 ]

| 구성 요소 | 설명 | 예시 |
|---|---|---|
| `extends Error` | Error 클래스를 상속하여 커스텀 에러 생성 | `class MyError extends Error { ... }` |
| `super(message)` | 부모(Error)의 생성자를 호출하여 message 초기화 | `super('에러 메시지')` |
| `this.name` | 에러 이름 지정 (미지정 시 'Error'로 표시됨) | `this.name = 'MyError'` |
| 추가 프로퍼티 | 필요한 정보를 자유롭게 추가 가능 | `this.statusCode = 400` |
| `instanceof` | catch에서 커스텀 에러 종류를 구분할 때 사용 | `error instanceof MyError` |

[ 사용자 정의 예외 활용 패턴 ]

| 패턴 | 설명 | 사용 시점 |
|---|---|---|
| 단일 커스텀 에러 | Error를 직접 상속한 단일 클래스 | 간단한 커스텀 에러가 필요할 때 |
| 에러 계층 구조 | 베이스 커스텀 에러를 상속받은 세부 에러 클래스 | 에러를 체계적으로 분류해야 할 때 |
| 추가 프로퍼티 확장 | statusCode, field 등 추가 정보 포함 | API 에러, 폼 유효성 검사 등 |

```
class ValidationException extends Error{
    constructor(message, field){
        super(message);
        this.field = field;
    }
}
const inputs = [
    { type: 'input', name: 'memberId', value: 'ksmartid' },
    { type: 'input', name: 'memberEmail', value: 'ksmartid' },
];

const validationFn = (data) => {
    if(data.name === 'memberId'){
        if(!data.value) throw new ValidationException('아이디를 입력해주세요.', data.name)
    }else if(data.name === 'memberEmail'){
        if(!data.value || !data.value.includes('@')) throw new ValidationException('이메일 형식에 어긋납니다.', data.name)
    }
    return true;
}

try{
    inputs.forEach(input => validationFn(input));
    console.log('input 데이터 서버에 요청');
}catch(error){
    if(error instanceof ValidationException){
        console.log(`필드: ${error.field}, 메세지: ${error.message}`);
    }
}
```
