## 자바스크립트
- 웹 페이지에서 동작을 담당하고 있는 객체지향 스크립트 언어이다.
- 자바스크립트는 script의 태그를 열어 태그내부에 코드를 작성한다. 
- 자바스크립트는 외부 파일을 로드 할 수 있다.
- 자바스크립트 코드는 주로 하단에 배치를 한다.

#### 컴파일러 언어 vs 인터프리터 언어
컴파일러 언어
- 컴파일 타임에 소스코드 전체 한번에 머신 코드로 변환한 후 실행
- 실행 파일을 생성
- 컴파일 단계, 실행단계 분리
- 실행에 앞서 컴파일은 단 한번 수행
- 실행 속도가 빠름

인터프리터 언어
- 코드가 실행되는 단계인 런타임에 문 단위로 한 줄씩 바이트코드로 변환한 후 실행
- 실행파일을 생성하지 않음
- 인터프리터 단계와 실행단계 분리X 한 줄씩 변환하고 즉시 실행
- 코드가 실행될 때마다 인터프리터 과정 반복 수행
- 코드 실행 속도가 비교적 느림

#### 개발환경 - 크롬 개발자 도구
- 요소(Elements) : HTML(Element), 요소에 적용된 스타일(CSS)을 검사
- 콘솔(Console) : 자바스크립트 오류 체크, 디버깅
- 소스(Sources) : 웹 자원 출력, 오류 체크와 디버깅


## 변수
- 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 공간을 식별하기 위해 붙인 이름.
- 프로그램 언어에서 값을 저장하고 참조(값의 위치)하는 상징적인 이름.

#### 코드 작성시 유의사항
- 식별자는 특수문자를 제외한 문자, 숫자, 언더스코어(_), 달러 기호($)를 포함할 수 있다.
- 단, 식별자는 특수문자를 제외한 문자, 언더스코어(_), 달러 기호($)로 시작해야 한다.
- 숫자로 시작하는 것은 허용하지 않는다.
- 예약어는 식별자로 사용할 수 없다.

```
//var 1a; // 변수명을 선언시 숫자로 시작하는 변수명은 사용금지
var a1 = 100; // 불분명한 변수명으로 선언하지 않는다. -> 지양
var score1 = 100; // 변수명도 불명확 -> 지양
var userName = '사용자1';

// private 변수를 선언할 때 주로 _를 접두사로 활용
var _privateValue = 'private';
console.log(_privateValue);

// $ 주로 dom 객체를 가리키는 변수를 선언할 때 활용
var $h2 = document.getElementById('title');
console.log($h2);
console.log($h2.textContent);

// 예약된 키워드는 변수명으로 사용할 수 없다.
//var if; // if는 예약된 키워드이기에 선언할 수 없다. 
```

#### 변수선언
- 변수 선언시 var, let, const 키워드를 사용한다.
- 선언단계: 변수 이름을 등록해서 자바스크립트 엔진에 변수의 존재를 알림
- 초기화단계: 값을 저장하기 위한 메모리 공간을 확보하고 암묵적으로 undefined를 할당해 초기화.
- GC(garbage collector) :  애플리케이션이 할당한 메모리공간을 주기적으로 검사하여 더 이상 사용되지 않은 메모리를 해제하는 기능

var

```
// var : 재선언 가능, 재할당 가능
var age = 30;
console.log(age)

var age = 40;
console.log(age)
```

let

```
// let : 변경이 가능한 데이터를 가리키는 변수 선언키워드
// 재선언 불가, 재할당 가능
let a = 10;
console.log('a의 값 : ', a);
//let a = 20; // 재선언 불가
a = 20;
console.log('a의 값 : ', a);

// let은 선언만 작성해도 가능하다.
// 선언 후 초기화가 가능
let gisu;
console.log(gisu);
gisu = 58;
console.log(gisu);
```

const

```
// const : 변경이 불가한 데이터(상수, 객체, 함수)를 선언하는 변수키워드
/ 재선언 불가, 재할당 불가
const b = 100;
//const b = 200; // 재선언 불가
//b = 200; // 재할당 불가

// const 선언 시 무조건 초기화해야한다.
//const student;
//student = '학생';
const student = '학생';
```

#### 변수 호이스팅
- 변수 선언(선언단계와 초기화단계)이 소스코드가 순차적으로 실행되는 런타임 이전단계에서 먼저 실행되어 참조에러가 발생해야하지만 undefined로 출력되는 자바스크립트 고유의 특징.
- var: 선언과 초기화(undefined)가 호이스팅됨
- let, const: 선언만 호이스팅되고 초기화는 안됨 (TDZ - Temporal Dead Zone)
- TDZ (Temporal Dead Zone) : JavaScript에서 변수가 선언되었지만 아직 초기화되지 않아 접근할 수 없는 시간적 구간

```
console.log(className);
var className = 'ksmart301';
console.log(className);

// let, const로 선언된 변수는 변수 이전에 참조 불가능
//console.log(letValue); // TDZ 규칙에 어긋난 문법에러
let letValue = 'let으로 선언한 변수';
console.log(letValue);

// 블록스코프 규칙 준수 : let, const
// 함수스코프 규칙 준수 : var
const teacherName = '홍길동';
var teacherAge = 25;
{
    console.log(teacherName); // let, const도 호이스팅
    console.log(teacherAge);
    //const teacherName = '김철수';
    var teacherAge = 30;
}
```

#### 권장 사항
1. 기본적으로 const를 사용하고, 재할당이 필요한 경우에만 let 사용
2. var는 호이스팅과 스코프 문제로 인해 사용 지양
3. 변수명은 의미있고 명확하게 작성
4. 전역 변수 사용 최소화 (오염 방지)
5. 하나의 변수에는 하나의 값만 저장하는 것이 원칙

#### 값과 리터럴
- 값 : 표현식이 평가되어 실행되는 결과
- 리터럴 : 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기법 (값, 표현식)

#### 표현식과 문
- 표현식 : 값으로 평가될 수 있는 문
- 문 : 프로그램을 구성하는 기본 단위이자 최소 실행 단위
- 토큰 : 키워드, 식별자, 연산자, 리터럴, 세미콜론, 마침표 등 문법적 의미를 가지는 코드의 기본요소

#### 표기법
- 카멜 표기법 : 소문자로 시작하여 2번째 단어부터 첫글자가 대문자로 시작하는 것
- 파스칼 표기법: 대문자로 시작하며 2번째 단어부터는 첫글자가 대문자로 시작하는 것
- 스네이크 표기법: 모두 소문자로 표기, 단어와 단어사이에 언더바를 붙여 표기
- 헝가리안 표기법: 변수명의 앞에 자료형을 붙이는 것

##### 카멜 표기법
- 규칙: 첫 단어는 소문자, 이후 단어의 첫 글자는 대문자
- 용도: 변수명, 함수명, 메서드명

좋은 예시 (Good)
- userName, totalAmount, isActive, getUserList

나쁜 예시 (Bad)
- username (구분 없음), UserName (파스칼 표기법), user_name (스네이크 표기법)

##### 파스칼 표기법
- 규칙: 모든 단어의 첫 글자를 대문자로
- 용도: 클래스명, 생성자 함수명

좋은 예시 (Good)
- UserAccount, ShoppingCart, DatabaseConnection

나쁜 예시 (Bad)
- userAccount (카멜 표기법), User_Account (스네이크 표기법)

##### 스네이크 표기법
규칙: 모든 단어를 소문자로, 단어 사이는 언더바(_)로 연결
용도: 데이터베이스 컬럼명, 상수명 (대문자 버전)

```
const apiResponse = {
    result_type : 'JSON'
}
```

##### 상수명 표기법
- 규칙: 모든 단어를 대문자로, 단어 사이는 언더바(_)로 연결
- 용도: 상수 (변하지 않는 값)

```
// 상수 : 프로그램 종료시까지 유지되는 값
const API_KEY = '12345678';
const VITE_BASE_URL = 'http://localhost:8080';     
```

##### 헝가리안 표기법
- 규칙: 변수명 앞에 자료형을 나타내는 접두사 추가
- 용도: 과거에 사용되었으나 현재는 권장하지 않음

```
let numMemberNo = 1;
const response = {
    memberNo : 1
}
// 앞의 자료형을 변경 (ex : strMemberNo)
// 헝가리안 표기법을 사용하지 않는다.
numMemberNo = response.memberNo; 
```

#### 메서드 네이밍 규칙
get : 값을 가져온다 / getName() : 반환값이 있는 메서드  
set : 값을 설정한다 / setAge(25) : 객체의 필드에 값을 삽입  
is / has  : 상태를 확인한다 / isValid(), hasPermission() : boolean 형태의 값을 반환  
add / remove : 추가/제거한다 / addItem(), removeItem() : 데이터 누적/삭제  
to : 형 변환 / toString(), toJSON() : 특정 형으로 변환하여 반환  
create / delete : 생성/삭제 / createUser(), deleteUser() : 객체나 데이터 생성/삭제  
find / search : 검색한다 / findById(), searchUser() : 데이터 검색  
update : 갱신한다 / updateProfile() : 기존 데이터 수정


## 데이터 타입
- 자바스크립트의 모든 값은 데이터 타입을 갖음.
- 7개의 데이터 타입은  원시타입, 객체타입으로 분류.

원시 타입(Primitive)
- String : 문자열 / 'Hello', "World"
- Number : 숫자 (정수, 실수) / 42, 3.14
- Boolean : 논리값 / true, false
- Null : 의도적인 빈 값 / null
- Undefined : 값이 할당되지 않음 / undefined

참조 타입(Reference)	
- Object : 객체, 배열, 함수 등 / {}, [], function(){}

#### 문자형 (String)
- 텍스트 데이터를 표현
- 작은따옴표(' '), 큰따옴표(" "), 백틱(` `) 사용
- 백틱은 템플릿 리터럴로 변수 삽입 가능

```
let strValue1 = '한국스마트정보교육원'; // 권장 good
let strValue2 = "한국스마트정보교육원"; // bad
let strValue3 = `한국스마트정보교육원`; // 변수를 활용하지 않을 경우 bad
console.log(strValue1, strValue2, strValue3);

// 백틱(템플릿 표기법) vs 문자열 연결(+)
let name = '홍길동';
let age = 20;

console.log('이름 : '+name+", "+"나이 : "+age+"세");
console.log(`이름 : ${name}, 나이 : ${age}세`);

// 여러줄 문자열(템플릿표기법)
const multiLine = `첫번째줄
두번째줄
세번째줄
`;
let multiLine2 = '첫번째줄\n두번째줄\n세번째줄';
console.log(multiLine);
console.log(multiLine2);

// 문자열인식('', "")
// 이름: "홍길동"
console.log(`이름: "${name}"`);

// 형변환
let convertStr = String(20); // good
console.log(convertStr); 
console.log(20); // bad
```

#### 숫자형 (Number)
- 정수와 실수를 모두 표현
- 특별한 값: Infinity, -Infinity, NaN
- 2진수, 8진수, 16진수 표현 가능

```
let num1 = 10;
let num2 = -10;
let num3 = 10.001;
let num4 = -10.001;
let num5 = 10 / 0;
let num6 = 10 / -0;
let num7 = 10 * "가";
console.log(num1);
console.log(num2);
console.log(num3);
console.log(num4);
console.log(num5); // Infinity
console.log(num6); // -Infinity
console.log(num7); // NaN

// 형변환
let convertNum = Number('100');
console.log(convertNum);
convertNum = Number('가나다');
console.log(convertNum);

// parseInt, parseFloat
console.log(parseInt(10.1234));
console.log(parseFloat('123.1234'));
```

#### 논리형 (Boolean)
- true 또는 false 두 가지 값만 가짐
- 조건문에서 많이 사용
- 다른 타입의 값을 논리형으로 변환 가능

```
let isBoolean1 = true;
let isBoolean2 = false;
let isBoolean3 = 1 >= 2;
console.log(isBoolean3);

// 형변환
console.log(Boolean(0)); // false
console.log(Boolean(1)); // true
console.log(Boolean(-1)); // true
console.log(Boolean('')); // false
console.log(Boolean(' ')); // true
console.log(Boolean('가')); // true
console.log(Boolean(undefined)); // false
console.log(Boolean(null)); // false
console.log(Boolean([])); // true
console.log(Boolean({})); // true
console.log(Boolean(function(){})); // true
```

#### null
- '값이 없음'을 의도적으로 명시
- 개발자가 명시적으로 할당
- "이 변수는 비어있다"라고 표현할 때 사용

주의! typeof null은 'object'를 반환
- 이것은 JavaScript의 초기 버그지만 호환성을 위해 수정되지 않았음.
- null 확인은 === 연산자를 사용.

#### undefined
- 변수가 선언되었지만 값이 할당되지 않은 상태
- JavaScript 엔진이 자동으로 할당
- 개발자가 의도적으로 할당하지 않음 (null 사용)

#### typeof 연산자
- typeof 연산자로 값의 자료형을 확인할 수 있습니다.
- 문법: typeof 값 → 자료형을 문자열로 반환

```
console.log(typeof '문자열'); // string
console.log(typeof 100); // number
console.log(typeof NaN); // number
console.log(typeof true); // boolean
console.log(typeof undefined); // undefined
console.log(typeof null); // object
console.log(typeof []); // object
console.log(typeof {}); // object
console.log(typeof function(){}); // function
```
