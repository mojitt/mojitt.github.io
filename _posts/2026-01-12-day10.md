#### 변환 속성
변환이란, 현재 요소의 상태에서 크기, 기울임, 회전, 위치 등을 변화 시킬 수 있는 것을 의미합니다.

**1.translate 속성값**
```
<head>
    <style>
        .box1{
            padding: 20px 0px;
            border: 1px solid black;
            margin: 5px 0px;
        }
        .box1 div{
            width: 100px;
            height: 100px;
            background-color: yellow;
            transform: translate(100px, 20px);
        }
    </style>
</head>
<body>
    <div class="box1">
        <div></div>
    </div>
</body>
```
transform : translate(x좌표 [, y좌표]) 
- 요소의 현재위치에서 다른 곳으로 이동할 수 있습니다. 
- translate 속성값 괄호에 첫번째 인자값은 x 축 이동좌표값을 의미합니다. 
- translate 속성값 괄호에 두번째 인자값은 y 축 이동좌표값을 의미하며 필수는 아닙니다.

\* y축 생략시 x축으로만 이동합니다. <br>
\* x축 양수는 오른쪽으로, 음수는 왼쪽으로 이동합니다. <br>
\* y축은 양수는 아래로, 음수는 위로 이동합니다. <br>

transform: translateX() -> x축으로만 이동합니다. <br>
transform: translateY() -> y축으로만 이동합니다. 

**2.scale 속성값**
```
<head>
    <style>
        .box3{
            padding: 50px;
            border: 1px solid black;
            margin: 5px 0px;
        }
        .box3 div{
            width: 100px;
            height: 100px;
            background-color: yellowgreen;
        }
        .box3 div:hover{
            transform: scale(1.5, 1);
        }
    </style>
</head>
<body>
    <div class="box3">
        <div></div>
    </div>
</body>
```
transform : scale(너비 [, 높이])
- 현재 요소의 크기를 제어하는 속성값 입니다.
- scale 값은 배수 단위 인자값을 지정합니다. 
- 만약 인자값을 하나만 작성한다면 가로, 세로 크기 배수가 됩니다.
- 인자값을 2개 모두 작성한다면 첫번째는 가로, 두번째는 세로 크기값이 됩니다.

\* 크기 조절 축이 요소의 정중앙이 됩니다.

transform: scaleX() -> 가로 크기 배수 값 <br>
transform: scaleY() -> 세로 크기 배수 값

**3.ratate 속성값**
```
<head>
    <style>
        .box5{
            padding: 50px 30px;
            border: 1px solid black;
        }
        .box5 div{
            width: 100px;
            height: 100px;
            background-color: khaki;
            margin-top: 30px;
        }
        .rBtn:active~div{
            transform: rotate(50deg);
        }
        .lBtn:active~div{
            transform: rotate(-80deg);
        }
    </style>
</head>
<body>
    <div class="box5">
        <button class="rBtn">시계방향</button>
        <button class="lBtn">반시계방향</button>
        <div>회전</div>
    </div>
</body>
```
transform: rotate(회전각도값 deg)
- 현재 요소의 시계 또는 반시계 방향으로 회전을 할 수 있는 속성값입니다.

**4.skew 속성값**
```
<head>
    <style>
        .box6{
            padding: 100px;
            border: 1px solid black;
        }
        .box6 div{
            width: 100px;
            height: 100px;
            background-color: skyblue;
            margin-top: 30px;
        }
        .xBtn:active~div{
            transform: skew(-30deg, 0deg);
        }
        .yBtn:active~div{
            transform: skew(0deg, 30deg);
        }
    </style>
</head>
<body>
    <div class="box6">
        <button class="xBtn">x축 기울임</button>
        <button class="yBtn">y축 기울임</button>
        <div>기울임</div>
    </div>
</body>
```
transform: skew(x축각도 [, y축각도])
- 현재 요소의 위치에서 x축 또는 y축으로 기울임을 제어하는 속성값입니다.
- 요소를 가로로 기울이거나 세로로 기울일 수 있습니다. 
- 첫번째 인자는 x축 각도를 의미하며, 양수 입력시 왼쪽으로 기울고 음수 입력시 오른쪽으로 기웁니다. 
- 두번째 인자는 y축 각도를 의미하며, 양수 입력시 위쪽으로 기울고 음수 입력시 아래로 기웁니다. 


#### 전환 속성
전환이란, 요소의 상태가 변화된느 과정을 의미합니다.

**transition 속성**
```
<head>
    <style>
        .box1{
            padding: 20px;
            border: 1px solid black;
            transition: all 2s;
            margin: 5px 0px;
        }
        .box1:hover{
            background-color: orangered;
            color: white;
        }
    </style>
</head>
<body>
    <div class="box1">
        마우스 오버하면 배경색이 서서히 바뀔 거에요.
    </div>
</body>
```
transition: (전환하고자 하는 속성명1) (시간), (전환하고자 하는 속성명2) (시간), ... 
- 해당 요소를 마우스 오버시 변환 되는 과정을 보여줄 수 있습니다.
- 모든 속성을 전환하고자 할때는 all (시간)

## JAVA
#### 자바의 특징
- 이식성이 높은 언어
- 객체 지향 언어(OOP:Object Oriented Programming) : 부품 객체를 먼저 만들고, 이것들을 조합해 전체 프로그램을 완성하는 기법
- 자바는 처음부터 OOP 개발용 언어로 설계
- 캡슐화, 상속, 다형성 기능 완벽하게 지원
- 함수적 스타일 코딩 지원
- 메모리를 자동으로 관리
- 다양한 애플리케이션 개발 가능
- 멀티 스레드 쉽게 구현 가능
- 동적 로딩(Dynamic Loading) 지원
- 막강한 오픈 소스 라이브러리 풍부

#### 자바 프로그램 개발 순서
1. .java 소스 파일 작성
2. 컴파일러(javac.exe)로 바이트 코드 파일(.class) 생성
3. JVM 구동 명령어(java.exe)로 실행

#### 자바 작성 규칙
1. 자바에서는 변수에 값을 초기화 하지 않으면 사용할 수 없다.
2. 예약어는 변수명이 될수 없다.
3. 변수 선언시 존재하는 데이터 타입을 명시해야한다.
4. 변수명은 숫자로 시작할 수 없습니다.
5. 변수명에 $,_ 제외한 특수문자는 포함 될 수 없습니다.
6. 변수명은 대소문자를 구분하므로 a와 A 변수는 다른 저장소입니다.


## 데이터 타입

#### 정수형 데이터 타입
1. 기본타입
- 정수, 실수, 문자, 논리

2. 참조타입
- 문자열, 배열, 객체

종류 : byte, short, int, long
- byte : 1byte -> 8bit -> 1bit(부호) / 7bit(값) / -128~127
- short : 2byte -> 16bit -> 1bit(부호) / 15bit(값) / -32768~32767
- int : 4byte -> 32bit -> 1bit(부호) / 31bit(값)
- long : 8byte -> 64bit
- 담을 수 있는 숫자의 크기는 byte < short < int < long

리터럴 표기법
- 자바에서는 리터럴도 데이터 타입을 가지며 정수형은 기본 int 타입입니다. <br>
이때 int로 표현할 수 없는 값을 리터럴로 표기하는 경우
컴파일 오류가 발생하며 뒤에 l 또는 L 키워드를 붙여 주셔야 
long 타입으로 변환되어 리터럴 표기가 가능해집니다.
- 자바에서 문자 리터럴 표기시 작은 따옴표를 사용하여 표기합니다.
- 유니코드는 0부터 시작하므로 char 타입에 음수값을 대입할 수 없습니다.

#### 실수형 데이터 타입
소수점을 갖는 숫자형식을 의미합니다. <br>
자바에서 float, double 두가지의 타입으로 실수를 표기합니다.

종류 : float, double
- float(4byte, 32bit) -> 1bit(부호부) / 8bit(지수부) / 23bit(가수부)
- double(8byte, 64bit) -> 1bit(부호부) / 11bit(지수부) / 52bit(가수부)
- 부호부 : +, -
- 지수부 : 소수점 자릿수, 10의 자리수
- 가수부 : 값
- 기본타입 : double

\* 자바에서는 실수에 대하여 정확 값이 아닌 부동소수점을 사용한 얼추 그쯤에 대한 값을 다룬다. <br>
\* 부동소수점은 정확값을 갖지 않으므로 정확한 연산에서는 사용을 권장하지는 않는다.

리터럴 표기법
- 실수 리터럴은 기본 데이터 타입이 double이기 때문에 float 타입 변수에 대입시 오류를 발생합니다.
- 이때 리터럴 끝에 f 또는 F를 명시하면 float으로 변환하여 대입할 수 있습니다.

#### 논리형 데이터 타입
자바에서 참과 거짓을 의미하는 데이터 타입을 말합니다.

boolean 타입 하나만 사용합니다. 1byte 저장 공간을 갖습니다.
- 리터럴 값 : true(참), false(거짓)
- 주로 연산식의 결과 값으로 반환 되어지는 데이터 타입입니다.

#### 문자열 데이터 타입
자바에서 하나이상의 문자들 그룹을 의미하는 데이터 타입입니다. <br>
String 타입 하나만 사용합니다. 리터럴 표기시 ""(큰따옴표)를 사용하여 표기합니다.

\* 자바 13이후 버전에서는 """ """ 을 사용하여 문자열 표기시 줄바꿈을 유지할 수 있습니다.


## 형변환
#### 자동형변환
서로 타입이 다른 값을 변수에 대입할 대 별도의 표기 없이 자동으로 대입되는 것을 의미합니다. <br>
작은 데이터 큰 데이터타입변수에 대입하는 경우 자동 형변환이 됩니다.
- byte < short < int < long < float < double

#### 강제형변환
문법) (변환타입)값을 담은 변수 <br>
더 큰 그릇에 있는 값을 더 작은 그릇에 대입할 때 개발자가 데이터의 문제가 없으므로 강제로 형변환을 시키는 방법입니다.

특징
- 작은 1byte 크기의 byte 타입에 대입할 경우 컴파일러가 논리상 문제가 발생할 수 있다고 오류를 내고 개발자가 실제 저장값에 문제가 없다고 강제로 형변환 합니다.
- 실수를 정수에 강제형변환하면 소수점을 버립니다.

#### 연산식에서의 형변환
연산식에서의 자동형변환 1
- 연산식에서 피연산들 중 더 큰 데이터 타입으로 자동 형변환 하여 연산하게 됩니다.

연산식에서의 자동형변환 2
- 피연산자의 데이터 타입이 같은 경우 정수는 int, 실수는 double로 연산의 결과를 반환합니다.

특징
- byte 변수 + byte 변수의 결과는 int로 자동형변환 되므로 byte 담을 수 없습니다.
- 연산의 자동형변환 문제 때문에 정수는 보통 int 타입을 많이 사용합니다.
- double+double은 double이므로 float에 담을 수 없습니다.
- int/int의 결과는 int이므로 소수점을 버립니다. 실수 연산을 원하면 한쪽의 피연산자를 강제로 double로 형변환 합니다.
