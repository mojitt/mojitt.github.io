## Lifting State Up
#### Shared State
- 하위 컴포넌트가 공통된 부모 컴포넌트의 State를 공유하여 사용하는것

#### State 끌어올리기
- 하위 컴포넌트의 State를 공통된 부모 컴포넌트로 끌어올려서 공유하는 방식

```
//Parent

import { useState } from "react"
import ChildCompo from "./ChildCompo"
import InterfaceCompo from "./InterfaceCompo"

const ParentCompo = () => {

    const [txt,setTxt] = useState("")

    const handlerChange = (e) => {
        setTxt(e.target.value)
    }

    return (
        <div>
            <h2>아래 텍스트를 입력해보세요.</h2>
            <input type="text" value={txt} onChange={handlerChange} />
            <ChildCompo value={txt} />
            <InterfaceCompo value={txt} />
        </div>
    )

}
export default ParentCompo
```

```
//Child
const ChildCompo = (props) => {

    return (
        <div>

            {props.value}

    )

}
export default ChildCompo
```

```
//Interface
import ChildCompo from "./ChildCompo"

const InterfaceCompo = (props) => {

    return (
        <div>
            <h4>나는 그냥 통과하는 컴포넌트 {txt} </h4>
            <ChildCompo value={props.value} />
        </div>
    )

    /*
    리액트에서는 props를 사용하여 하위 컴포넌트로 데이터를 전달할 수 있습니다.
    하지만 데이터를 필요로하지 않는 중간 컴포넌트에서도 굳이 받아서 전달해야하는 경우가 발생하게 됩니다.
    
    리액트에서 context라는 기능을 제공하여 context에 포함된 모든 컴포넌트에서 쉽게 데이터를 읽을 수 있도록 제공하고 있습니다.
    *context : 공유 데이터를 등록해두는 요소
     */
}
export default InterfaceCompo
```

## Context
#### 컨텍스트란?
- 컴포넌트들 사이에서 데이터를 props를 통해 전달하는 것이 아닌, 컴포넌트 트리를 통해 곧바로 데이터를 전달하는 방식
- 어떤 컴포넌트든지 컨텍스트에 있는 데이터에 쉽게 접근할 수 있음

#### 언제 사용하는지?
- 여러 컴포넌트에서 계속해서 접근이 일어날 수 있는 데이터들이 있는 경우
- Provider의 모든 하위 컴포넌트가 얼마나 깊이 위치해 있는지 관계없이 컨텍스트의 데이터를 읽을 수 있음

#### 컨텍스트 사용 전 고려할 점
- 컴포넌트와 컨텍스트가 연동되면 재사용성이 떨어짐
- 다른 레벨의 많은 컴포넌트가 데이터를 필요로 하는 경우가 아니라면, 기존 방식대로 props를 통해 데이터를 전달하는 것이 더 적합

#### 컨텍스트 API
- React.createContext()
	- 컨텍스트를 생성하기 위한 함수
	- 컨텍스트 객체를 리턴함
	- 기본값으로 undefined를 넣으면 기본값이 사용되지 않음
- Context.Provider
	- 모든 컨텍스트 객체는 Provider라는 컴포넌트를 갖고 있음
	- Provider 컴포넌트 하위 컴포넌트들을 감싸주면 모든 하위 컴포넌트들이 해당 컨텍스트의 데이터에 접근할 수 있게됨
	- Provider에는 value라는 prop이 있으며, 이것이 데이터로써 하위에 있는 컴포넌트들에게 전달됨
	- 여러 개의 Provider 컴포넌트를 중첩시켜 사용할 수 있음
- Class.contextType
	- Provider 하위에 있는 클래스 컴포넌트에서 컨텍스트의 데이터에 접근하기 위해 사용
	- 단 하나의 컨텍스만을 구독할 수 있음
- Context.Consumer
	- 컨텍스트의 데이터를 구독하는 컴포넌트
	- 데이터를 소비한다는 뜻에서 consuming 컴포넌트라고도 부름
	- consuming 컴포넌트는 컨텍스트 값의 변화를 지켜보다가 값이 변경되면 재렌더링됨
	- 하나의 Provider 컴포넌트는 여러 개의 consuming 컴포넌트와 연결될 수 있음
	- 상위 레벨에 매칭되는 Provider가 없을 경우 기본값이 사용됨
- Context.displayName
	- 크롬의 리액트 개발자 도구에서 표시되는 컨텍스트 객체의 이름

#### 여러 개의 컨텍스트 사용하기
- Provider 컴포넌트와 consumer 컴포넌트를 여러 개 중첩해서 사용하면 됨

#### useContext()
- 함수 컴포넌트에서 컨텍스트를 쉽게 사용할 수 있게 해주는 훅
- React.createContext() 함수 호출로 생성된 컨텍스트 객체를 인자로 받아서 현재 컨텍스트의 값을 리턴
- 컨텍스트의 값이 변경되면 변경된 값과 함께 useContext() 훅을 사용하는 컴포넌트가 재렌더링됨

```
//Parent
import { useState } from "react"
import ChildCompo from "./ChildCompo"
import InterfaceCompo from "./InterfaceCompo"
import { useContext } from "react"
import { SampleContext } from "./SampleContext"
import { MultiContext } from "./MultiProvider"

const ParentCompo = () => {

    const [txt,setTxt] = useState("")
    //const data = useContext(SampleContext)
    //const contextData = useContext(MultiContext)

    const {data, changeData} = useContext(SampleContext)

    const handlerClick = () => {
        changeData(txt)
    }

    const handlerChange = (e) => {
        setTxt(e.target.value)
    }

    return (
        <div>
            <h2>아래 텍스트를 입력해보세요. / {data} </h2>
            <input type="text" value={txt} onChange={handlerChange} />
            <button onClick={handlerClick} style={{margin:"5px"}}>변경!</button>
            <ChildCompo value={txt} />
            <InterfaceCompo value={txt} />
        </div>
    )

}
export default ParentCompo
```
