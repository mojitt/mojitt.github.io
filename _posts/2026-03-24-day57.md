## jQuery — 선택자 · 이벤트 · DOM 조작
jQuery란?
- 경량화·호환성·애니메이션 효과 등을 포함한 JavaScript 라이브러리.
- CSS 선택자를 활용해 DOM 객체를 쉽게 탐색·조작할 수 있도록 설계됨.
- 순수 JavaScript 기반이므로 반드시 외부 파일을 로드한 뒤 사용해야 함.

- 핵심 함수: jQuery() == $() — 완전히 동일한 함수.

[ jQuery 로딩 방식 비교 ]

| 방식 | 코드 | 특징 |
| :--- | :--- | :--- |
| 다운로드 | `<script src="js/jquery-3.7.1.js"></script>` | 서버에 파일 보관. 오프라인 동작 가능. 직접 관리 필요. |
| CDN | `<script src="https://cdnjs.../jquery.min.js"></script>` | 별도 파일 불필요. 인터넷 필수. 캐시 공유 효과. |

#### 직접 선택자
- $('CSS 선택자') 형태로 호출하면 jQuery 객체가 반환됨.
- 대상 요소가 없어도 빈 jQuery 객체가 반환되므로 null 체크가 필요 없음.
- 반환된 jQuery 객체에 체이닝으로 메서드를 연결할 수 있음.

- $('body') → jQuery 객체 (jQuery 메서드 사용 가능)
- $('body')[0] → 일반 HTMLElement (JS DOM API 사용 가능)

[ jQuery 객체 특징 ]

| 구분 | 예시 | 반환값 |
| :--- | :--- | :--- |
| 대상 있음 | `$('body')` | 해당 요소를 담은 jQuery 객체 |
| 대상 없음 | `$('body123')` | 빈 jQuery 객체 (length: 0) — null 아님 |
| JS 요소로 변환 | `$('body')[0]` | HTMLElement — JS DOM API 접근 가능 |
| 복수 선택 | `$('.myBtn')` | 조건에 맞는 모든 요소를 담은 jQuery 객체 |

```

#### $(this) vs e.target — 이벤트 대상 구분
- 이벤트 핸들러 내부에서 이벤트가 걸린 대상과 실제 클릭된 대상이 다를 수 있음.
- 특히 이벤트 위임 패턴에서 구분이 매우 중요함.

[ $(this) vs e.target vs e.currentTarget 비교 ]

| 구분 | 설명 | 사용 시점 |
| :--- | :--- | :--- |
| `$(this)` | 이벤트가 바인딩된 요소. 직접 등록 시 클릭 요소와 동일. | 직접 이벤트 등록 시 |
| `e.target` | 실제로 클릭(이벤트 발생)된 요소. 자식 요소 클릭 시 자식을 가리킴. | 이벤트 위임 시 실제 클릭 대상 특정 |
| `e.currentTarget` | 이벤트가 바인딩된 요소. `$(this)` 와 동일. | 화살표 함수에서 this 사용 불가할 때 |

```
$('.exBtn').click(function(e){
    const btnTxT = $(this).text();
    let jqObj;
    switch(btnTxT){
        case 'this':
            jqObj = $(this);
            break;
        case 'e.target':
            jqObj = $(e.target);
            break;
        default:
            jqObj = $(e.currentTarget);
            break;
    }
    $('#eventResult').text(`${btnTxT} : ${jqObj.text()} 버튼 (${e.eventPhase})`);
});
```

#### 인접 관계 선택자
- 선택된 요소를 기준으로 부모 / 자식 / 형제 / 위치를 탐색하는 메서드.
- 체이닝으로 연결하여 복잡한 DOM 구조에서도 원하는 요소를 정확히 찾을 수 있음.

[ 인접 관계 선택자 메서드 ]

| 분류 | jQuery | JS 대응 | 설명 |
| :--- | :--- | :--- | :--- |
| **부모 탐색** | `.parent()` | parentElement | 직접 부모 1개만 반환 |
| | `.parents('선택자')` | — | 조상 요소 중 선택자에 해당하는 전체 반환 |
| | `.closest('선택자')` | closest() | 조상 중 가장 가까운 노드만 반환 |
| **자식 탐색** | `.children()` | children | 직접 자식 전체 반환 |
| | `.find('선택자')` | querySelectorAll() | 자손 전체에서 선택자에 해당하는 요소 반환 |
| **형제 탐색** | `.prev()` | previousElementSibling | 이전 형제 요소 |
| | `.next()` | nextElementSibling | 다음 형제 요소 |
| | `.siblings()` | — | 형제 전체 반환 (자신 제외) |
| **위치 탐색** | `.first()` | — | 검색된 집합에서 첫 번째 |
| | `.last()` | — | 검색된 집합에서 마지막 |
| | `.eq(n)` | `[n]` | 검색된 집합에서 n번째 (0-index) |  

parent() vs parents() vs closest() 차이
- .parent() — 바로 위 부모 1개만. 선택자 인수 없음.
- .parents('ul') — 위쪽 조상 중 ul 전부 반환 (복수 가능).
- .closest('ul') — 위쪽 조상 중 가장 가까운 ul 1개만 반환. 이벤트 핸들러에서 행 탐색 시 가장 많이 사용.

```
$('.liBtn').click(e => {
    const $btn = $(e.target);
    const $closestUl = $btn.closest('Ul');
    const $li = $btn.parent();
    const $li2 = $closestUl.children('.li2');
    const lines = [
        `parent() => li.${$li[0].className}`,
        `parents('ul') => ${$btn.parents('ul').length}개`,
        `closest('ul') => ul.${$closestUl[0].className}`,
        `children() => ${$closestUl.children().length}개`,
        `children(.li2) => li.${$closestUl.children('.li2')[0].className}`,
        `find('.li1') => li.${$closestUl.find('.li1')[0].className}`,
        `prev() => li.${$li2.prev()[0].className}`,
        `next() => li.${$li2.next()[0].className}`,
        `siblings() => li.${$li2.siblings().length}개`,
        `first() => li.${$closestUl.children().first()[0].className}`,
        `last() => li.${$closestUl.children().last()[0].className}`,
        `eq(1) => li.${$closestUl.children().eq(1)[0].className}`
    ]

    $('#relResult').text(lines.join('\n'));

});
```

#### 배열 관련 메서드
- jQuery는 검색된 요소 집합을 순회하거나 변환하는 메서드를 제공.
- JS 배열 메서드와 유사하지만 동작 방식에 차이가 있으므로 구분이 필요.

[ jQuery 배열 관련 메서드 vs JS ]

| 메서드 | 설명 | 중단 여부 | JS 대응 |
| :--- | :--- | :--- | :--- |
| `$(arr).each(fn)` | jQuery 객체 요소 순회. `return false` 로 중단 가능. | return false 중단 | `forEach` (중단 불가) |
| `$.each(arr, fn)` | 일반 배열/객체 순회. 동일하게 `return false` 중단. | return false 중단 | `for...of` |
| `$(arr).map(fn)` | `return` 한 값만 새 jQuery 객체에 담아 반환. `undefined` 제외. | — | `map` (undefined 포함) |
| `$.grep(arr, fn)` | 조건을 만족하는 요소만 걸러서 일반 배열로 반환. | — | `filter` |  

each의 return true / false 의미
- return true → 계속 진행 (기본값, 생략 가능)
- return false → 순회 즉시 중단 (JS break와 동일 효과)
  
jQuery map vs JS map 차이
- JS map: return 값이 없으면 undefined가 배열에 포함됨.
- jQuery map: return 값이 없는 요소는 결과에서 제외됨.

#### 속성 조작 — attr / prop / val / data
- HTML 요소의 속성을 다루는 메서드는 4가지로 구분됨.
- 어떤 메서드를 써야 할지 혼동하기 쉬우므로 선택 기준을 명확히 이해해야 함.

[ attr / prop / val / data 선택 기준 ]

| 메서드 | 대상 | JS 대응 | 예시 |
| :--- | :--- | :--- | :--- |
| `.attr()` | HTML 마크업 속성 (`href`, `src`, `name`, `data-*` 등) | `getAttribute` / `setAttribute` | `$('a').attr('href')` |
| `.prop()` | 상태 속성 (`checked`, `selected`, `disabled` 등) | `element.checked` 등 | `$(':checkbox').prop('checked')` |
| `.val()` | form 요소 값 (`input`, `select`, `textarea`) | `element.value` | `$('#myInput').val()` |
| `.data()` | jQuery 내부 캐시 (DOM과 완전히 독립) | `dataset` (동작 방식 다름) | `$('#el').data('userId')` |

#### attr / removeAttr — HTML 마크업 속성
- attr vs val — 핵심 차이 (자주 혼동)
- .attr('value') → HTML에 작성된 초기값을 항상 반환 (사용자가 바꿔도 초기값 반환)
- .val() → 현재 입력된 실시간 값을 반환

- checked, selected, disabled 같은 상태 속성은 반드시 prop() 사용.
- .attr('checked')는 초기 상태만 반영하므로 동적 제어에 부적합.

```
const $attrResult = $('#attrResult');
$('#getAttrBtn').click(function(){
    $attrResult.text($('#attrInput').attr('value'));
});
$('#getValBtn').click(function(){
    $attrResult.text($('#attrInput').val());
});
$('#setAttrBtn').click(function(){
    $('#attrInput').attr('value', 'ksmart58');
});
$('#disableBtn').click(function(){
    $('#attrInput').attr('disabled', 'disabled');
});
$('#enableBtn').click(function(){
    $('#attrInput').removeAttr('disabled');
});
```

#### prop — 상태 속성 조작
- checked, selected, disabled 등 현재 상태를 나타내는 속성은 반드시 .prop()로 조작.
- .attr('checked')는 초기 마크업 상태만 반영하므로 동적 제어에 사용하면 안 됨.

```
$('#allCheck').click(e => $('.checks').prop('checked', $('#allCheck').prop('checked')));

$('.checks').click(e => {
    $('#allCheck').prop(
        'checked',
        $('.checks:checked').length === $('.checks').length
    );
});

$('#select').val('option2');
$('#select option').eq(2).prop('selected', 'optino3');
$('#select').prop('disabled', true);
```

#### data() vs dataset — jQuery 캐시 vs DOM 속성
- dataset (JS): 실제 HTML data-* 속성과 직접 연동. 변경 시 DOM에 반영됨.
- .data() (jQuery): jQuery 내부 캐시 객체에 저장. DOM과 완전히 독립. HTML에 반영되지 않음.

- 서버에서 내려온 data-* 값 읽기는 .attr() 사용, 클라이언트 전용 임시 데이터 저장은 .data() 권장.

```
$('#dataInput').data('userId', 'ksmartid');
console.log($('#dataInput').data('userId'));
//$('#dataInput')[0].dataset.userId = 'ksmartid';
console.log($('#dataInput')[0].dataset); 
```

        if(!value){
            //const cate = $(`label[for="${id}"]`).text();
            const cate = $(elem).prev().text();
            $(elem).next().text(`${cate} 필수입력항목입니다.`);
            $(elem).next()[0].classList.add('show');
            if(isFirst){
                $(elem).focus();
                isFirst = false;
            }
        } else {
            print.push(`id: ${id}, value: ${value}`);
        }
    });
    if($formEle.length === print.length){
        $('#valResult').text(print.join('\n'));
    }
});
```

#### 클래스 조작 — addClass / removeClass / hasClass / toggleClass
[ 클래스 조작 메서드 ]

| jQuery | JS 대응 | 설명 |
| :--- | :--- | :--- |

#### 요소 추가 / 삽입 / 제거
- jQuery로 요소를 동적으로 생성하고 원하는 위치에 삽입할 수 있음.
  
삽입 위치
- before ← [ prepend ← 대상 요소 → append ] → after

[ 요소 추가/삽입/제거 메서드 ]

| 메서드 | 위치 | JS 대응 |
| :--- | :--- | :--- |
| `.append(el)` | 선택 요소 내부 마지막 | `appendChild` |
| `.prepend(el)` | 선택 요소 내부 처음 | `insertBefore(el, firstChild)` |
| `.before(el)` | 선택 요소 앞 (형제) | `insertBefore` |
| `.after(el)` | 선택 요소 뒤 (형제) | `insertAdjacentElement('afterend')` |
| `.remove()` | 요소 자체를 DOM에서 완전 제거 | `remove()` |
| `.empty()` | 요소는 유지, 자식 노드만 전부 제거 | `innerHTML = ''` |
  
.remove() vs .empty() vs .html('') 차이
- .remove() → 선택된 요소 자체가 DOM에서 사라짐. 이벤트도 함께 제거됨.
- .empty() → 선택된 요소는 남고, 내부의 자식 노드만 모두 제거됨.
- .html('') → .empty()와 결과는 같지만 jQuery 내부 캐시(.data)가 제거되지 않을 수 있어 .empty() 권장.
  
jQuery 요소 생성 문법
- $('<태그 />', { 속성객체 }) 형태로 요소를 생성하고 속성을 한번에 설정 가능.

console.log($('<form></form>', { id: 'myForm', class: 'dynamic-form' }));

#### 체이닝 (Chaining)
- jQuery 메서드 대부분은 자기 자신(jQuery 객체)을 반환하므로 메서드를 연속으로 연결할 수 있음.
- 중간에 탐색 메서드를 쓰면 탐색 결과를 기준으로 이어서 메서드 호출 가능.
- 코드가 간결해지지만 과도한 체이닝은 가독성을 해칠 수 있으므로 적절히 사용.

- .end() — 체이닝 중 이전 탐색 결과로 복귀할 때 사용.

```
$('.chaining').css('background-color', 'hotpink')
                .find('input')
                .css('color', 'red') // div.chaining input
                .end() // 체이닝 중 이전 탐색 결과로 복귀 div.chaining
                .css('background-color', '');
```

#### 동적 이벤트 바인딩
- 동적으로 생성된 요소는 생성 당시 이미 등록된 이벤트 핸들러가 적용되지 않음.
- 이벤트 위임(Event Delegation)을 활용하면 동적 생성 요소에도 이벤트 적용 가능.

- 문법: $(정적_부모).on('이벤트', '대상_선택자', fn)

- $(document)에 위임하면 어디서나 동작하지만 성능에 불리.
- 가장 가까운 정적(고정) 부모 요소에 위임하는 것이 성능상 유리.

[ 일반 이벤트 vs 동적 이벤트 바인딩 비교 ]

| 방식 | 문법 | 동적 요소 적용 | 권장 여부 |
| :--- | :--- | :--- | :--- |
| 직접 등록 | `$('.btn').click(fn)` | 이후 생성된 요소엔 미적용 | 정적 요소에만 |
| document 위임 | `$(document).on('click', '.btn', fn)` | 적용됨 | 최후 수단 |
| 가까운 부모 위임 | `$('#wrap').on('click', '.btn', fn)` | 적용됨 | 권장 |

```
const num = (num => () => ++num)(0);
$('#addDynBtn').click(function(){
    const $btn = $('<button />', {type: 'button', class: 'btn dyn-btn'})
                                .css('margin', '4px')
                                .text(`버튼${String(num()).padStart(2,0)}`);
    $('#btnGroup').append($btn);
});

// 동적이벤트 바인딩 (동적으로 생성된 요소에 이벤트 등록)
//$(document).on(이벤트타입, 생성된 요소 css선택자, 이벤트 핸들러(fn));
//$(상위요소).on(이벤트타입, 생성된 요소 css선택자, 이벤트 핸들러(fn));
$('#btnGroup').on('click', '.dyn-btn', function(){
    $('#dynResult').text(`${$(this).text()} 클릭`);
});
```
