## 스타일 조작
#### 인라인 스타일 (style 프로퍼티)
- element.style.속성명으로 인라인 스타일을 읽거나 설정.
- CSS 속성명은 카멜케이스로 변환. 예: background-color → backgroundColor

```
const $ksmartTeam = document.querySelector('.ksmart-team');

$ksmartTeam.style.background = '#f8e9f6';
//$ksmartTeam.style.padding = '15px';
$ksmartTeam.style.paddingTop = '15px'; // 카멜표기법
$ksmartTeam.style.paddingBottom = '15px';
$ksmartTeam.style['padding-left'] = '15px'; // 케밥표기법 사용시
$ksmartTeam.style['padding-right'] = '15px';
$ksmartTeam.style.borderLeft = '4px solid #cc7fd5';
$ksmartTeam.style.margin = '15px 0';

console.log($ksmartTeam.style);
```

#### 클래스 조작 (classList)
- className: 클래스 전체를 문자열로 읽거나 설정. 기존 클래스 덮어쓸 위험 있음.
- classList: 개별 클래스를 안전하게 추가·삭제·변경 가능. 권장 방법.

[ classList 메서드 ]

| 메서드 | 설명 | 예시 |
| :--- | :--- | :--- |
| `add(class)` | 클래스 추가 (중복 추가 무시) | `classList.add('active')` |
| `remove(class)` | 클래스 삭제 | `classList.remove('active')` |
| `contains(class)` | 클래스 포함 여부 확인 (true/false) | `classList.contains('active')` |
| `replace(old, new)` | 클래스 교체 | `classList.replace('old', 'new')` |
| `toggle(class)` | 클래스 있으면 삭제, 없으면 추가 | `classList.toggle('active')` |
| `toggle(class, bool)` | true면 추가, false면 삭제 | `classList.toggle('active', isOn)` |
| `item(index)` | 인덱스로 클래스 반환 | `classList.item(0)` |

```
const $ksmartDev = document.querySelector('.ksmart-dev');
const classList = $ksmartDev.classList;

// 클래스 추가
classList.add('ks-58');
classList.add('ksmart-58');
classList.add('ks-58'); // 중복 추가 시 무시

// 클래스 삭제
classList.remove('ksmart-58');

// 클래스 포함여부
console.log(classList.contains('ksmart-58'), '<--클래스 포함여부'); 

// 클래스 교체
classList.replace('ks-58', 'ksmart-58');

// 반전 toggle(클래스) : 클래스가 존재한다면 삭제, 존재하지 않는다면 추가
classList.toggle('ks-58'); // 추가
classList.toggle('ks-58'); // 삭제

// toggle(클래스, true(추가) | false(삭제))
classList.toggle('ks-58', true);
classList.toggle('ks-58', 1 === 0);

// 인덱스번호 특정 클래스 반환
const secondClass = classList.item(1);
console.log(secondClass);

console.log($ksmartDev.classList);

// className은 전체문자열을 반환하고 클래스조작이 어려워,
// classList로 조작하는 것을 권장
$ksmartDev.className += ' ks-58';
console.log($ksmartDev.className);
```

## DOM 이벤트 (Event)
### 이벤트(Event)란?
- 브라우저는 처리해야 할 특정 사건이 발생하면 이를 감지하여 이벤트를 발생시킴.
- 클릭, 키보드 입력, 마우스 이동 등이 일어나면 특정 타입의 이벤트가 발생.
- 이벤트 핸들러(Event Handler): 특정 이벤트가 발생했을 때 호출될 함수.
- 이벤트 핸들러 등록: 특정 이벤트 발생 시 핸들러의 호출을 위임하는 것.

##### 이벤트 등록 방식

[ 이벤트 등록 방식 3가지 비교 ]

| 방식 | 등록 방법 | 핸들러 수 | 특징 / 주의점 |
| :--- | :--- | :--- | :--- |
| ① 어트리뷰트 방식 | `onclick="fn()"` | 1개 | HTML과 JS가 혼재 → 유지보수 어려움. 비권장. |
| ② 프로퍼티 방식 | `$el.onclick = fn` | 1개 | 재할당 시 이전 핸들러 덮어씀. `null` 할당으로 제거. |
| ③ addEventListener 방식 | `$el.addEventListener('이벤트', fn)` | 복수 | 동일 이벤트에 핸들러 여러 개 등록 가능. `removeEventListener` 로 특정 핸들러만 제거 가능. 권장. |

##### on 접두사 어트리뷰트
- 모든 HTML 태그는 on 접두사를 가진 이벤트 속성을 가짐.
- onclick, onchange, oninput 등 → 값으로 자바스크립트 코드 문자열을 직접 등록.
- 단, HTML과 JavaScript가 뒤섞여 관심사 분리(Separation of Concerns) 원칙에 어긋나므로 실무에서는 사용하지 않음.

```
function alertFn(elem){
    console.log(elem.textContent);
    alert(elem.textContent);
}

// Dom porperty 이벤트 핸들러 등록방식
const $proBtn = document.getElementById('proBtn');
$proBtn.onclick = function(){
    alert(this.textContent+'1');
    alertFn2(this);
}
function alertFn2(elem){
    alert(elem.textContent+'2');
}

// Dom addEventListener : 이벤트 핸들러를 등록하는 메소드 (중복 등록 가능)
const $methodBtn = document.getElementById('methodBtn');
$methodBtn.addEventListener('click', function(){
    alert(this.textContent);
});
$methodBtn.addEventListener('click', function(){
    alert(this.textContent);
});
function method(){
    alert(this.textContent);
}
$methodBtn.addEventListener('click', method);

// removeEventListener : 등록되어있는 이벤트 핸들러 제거하는 메소드
const $methodBtn2 = document.getElementById('methodBtn2');
$methodBtn2.addEventListener('click', function(){
    $methodBtn.removeEventListener('click', method);
});
```

### 이벤트의 종류

##### 마우스 이벤트

[ 마우스 이벤트 ]

| 이벤트 | 발생시점 | 비고 |
| :--- | :--- | :--- |
| `click` | 마우스 버튼을 클릭했을 때 | |
| `dblclick` | 마우스 버튼을 더블클릭했을 때 | |
| `mousedown` | 마우스 버튼을 누르는 순간 | |
| `mouseup` | 누르던 마우스 버튼을 뗄 때 | |
| `mousemove` | 마우스를 움직일 때 | 터치스크린에서 동작하지 않음 |
| `mouseover` | 마우스 포인터가 요소 위로 올라왔을 때 | 자식 요소 포함, 버블링 발생 |
| `mouseout` | 마우스 포인터가 요소 밖으로 나갔을 때 | 자식 요소 포함, 버블링 발생 |
| `mouseenter` | 마우스 포인터가 요소 안으로 들어왔을 때 | 자식 요소 미포함, 버블링 없음 |
| `mouseleave` | 마우스 포인터가 요소 밖으로 나갔을 때 | 자식 요소 미포함, 버블링 없음 |

```
document.getElementById('dblclick').addEventListener('dblclick', function(){
    alert('더블클릭발생');
});
document.getElementById('mouseover').addEventListener('mouseover', function(){
    alert('마우스오버발생');
});
```

##### 키보드 이벤트
- 키보드 이벤트 실행 순서: keydown → keypress → input → keyup
- ⚠️ keypress는 deprecated(폐기 예정) — keydown 또는 input으로 대체 권장.

[ 키보드 이벤트 ]

| 이벤트 | 발생 시점 | 비고 |
| :--- | :--- | :--- |
| `keydown` | 키를 누르는 순간 (모든 키) | 한글·특수키 포함, 실무에서 가장 많이 사용 |
| `keyup` | 누르던 키를 뗄 때 | |
| `keypress` | 키를 누르고 뗐을 때 | ⚠️ deprecated — 한글·특수키 미지원 |

```
/* keydown -> keypress -> input(사용자입력값 바인딩) -> keyup(사용자입력값 바인딩) */
const $input = document.getElementById('input');
$input.addEventListener('keydown', function(){
    console.log('keydown', '이벤트 발생', this.value);
});
$input.addEventListener('keypress', function(){
    console.log('keypress', '이벤트 발생', this.value);
});
$input.addEventListener('input', function(){
    //this.value = this.value.replace(/[^a-zA-Z]/g, ''); // 영어만 입력 받는 구문
    console.log('input', '이벤트 발생', this.value);
});
$input.addEventListener('keyup', function(){
    console.log('keyup', '이벤트 발생', this.value);
});
```

##### 포커스 이벤트
- focus / blur는 버블링이 발생하지 않음.
- 부모 요소에서 자식의 포커스 이벤트를 감지하려면 버블링이 있는 focusin / focusout을 사용.

[ 포커스 이벤트 ]

| 이벤트 | 발생 시점 | 버블링 |
| :--- | :--- | :--- |
| `focus` | 요소가 포커스를 얻었을 때 | X |
| `blur` | 요소가 포커스를 잃었을 때 | X |
| `focusin` | 요소가 포커스를 얻었을 때 | O |
| `focusout` | 요소가 포커스를 잃었을 때 | O |

```
const $focus = document.getElementById('focus');
$focus.addEventListener('focus', function(){
    console.log('focus 이벤트 발생');
});
$focus.addEventListener('blur', function(){
    console.log('blur 이벤트 발생');
});

*/
```

##### 폼(Form) 이벤트

[ 폼 이벤트 ]

| 이벤트 | 발생 시점 | 비고 |
| :--- | :--- | :--- |
| `input` | input / textarea 값이 변경될 때마다 즉시 | 매 입력마다 실시간으로 발생 |
| `change` | select / checkbox / radio 상태 변경, 또는 input 포커스 이탈 후 값이 달라졌을 때 | input과 달리 포커스 이탈 시점에 발생 |
| `submit` | 폼을 전송할 때 | `event.preventDefault()` 로 기본 전송 방지 가능 |
| `reset` | 리셋 버튼을 클릭할 때 | |

```
const $Id = document.getElementById('Id');
const $Room = document.getElementById('Room');
const $printBtn = document.getElementById('printBtn');
const $resultP = document.getElementById('resultP');

$Id.addEventListener('input', function(){
    console.log('$Id input 이벤트 발생');
});
$Id.addEventListener('change', function(){
    console.log('$Id change 이벤트 발생');
});
$Room.addEventListener('change', function(){
    console.log('$Room change 이벤트 발생');
});

$printBtn.addEventListener('click', function(){
    $resultP.textContent = `input: ${$Id.value}, select: ${$Room.value}`;
});
```
