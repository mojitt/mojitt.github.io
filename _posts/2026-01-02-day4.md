### 입력 양식 종류(2)
체크박스 입력 양식
```
<input type="checkbox" name="chk1" value="001">체크1
<input type="checkbox" name="chk1" value="002">체크2
```
- name 속성이 같으면 같은 그룹의 체크박스들이 됩니다.
- 각 체크박스 항목에 value 속성값을 작성해야 처리하는 곳에서 선택된 체크박스 값을 알 수 있습니다.
- checked 속성을 사용하여 초기 체크박스 선택을 할 수 있습니다.

파일 선택 입력 양식
```
<input type="file" name="uploadfile" multiple>
```
- 첨부파일 양식 필요시 사용합니다.
- multiple 속성 사용시 컨트롤키 누르고 여러 파일을 선택할 수 있습니다.

숫자 범위 입력 양식
```
<input type="range" onchange="alert(event.target.value)" min="0" max="10" step="2">
```
- 드래그하여 숫자를 조절할 수 있습니다
- min과 max 속성으로 최솟값과 최댓값을 설정할 수 있습니다.

여러줄 입력 가능한 텍스트 입력 양식
```
<textarea name="textA" cols="50" rows="10">초기값</textarea>
```
- cols 속성을 사용하여 초기 가로 길이를 제어할 수 있습니다.
- rows 속성을 사용하여 초기 세로 길이를 제어할 수 있습니다.
- 시작, 종료 태그 사이 입력된 값이 초기 텍스트가 됩니다.
- 이때, 초기 텍스트 앞뒤 공백이 값으로 들어가지 않게끔 주의합니다.

펼침 목록 입력 양식
```
<select name="select">
    <option value="item1">항목1</option>
    <option value="item2" selected>항목2</option>
    <option value="item3">항목3</option>
</select>
```
- select 태그 안에 option 태그를 사용하여 선택 목록을 내열합니다.
- option 태그에 value 속성이 없다면 시작종료태그 사이 내용이 전송됩니다.
- 있다면 value 속성값이 전송됩니다.(선택된 항목의) 
- option 태그에 selected 속성을 선언하면 초기 선택항목을 지정할 수 있습니다.

눈에 보이지 않는 입력 양식
```
<input type="hidden" name="hidden" value="1234">
```
- 사용자가 입력하지 않고 개발상 필요에 의해 숨겨서 데이터를 처리하는 곳으로 보내야하는 경우 사용합니다.
- value 속성에 입력된 값이 전송됩니다.

입력값 힌트부여 속성
```
<input type="text" placeholder="아이디를 입력하세요.">
<textarea placeholder="내용을 입력하세요."></textarea>
```
- placeholder 속성을 사용하여 텍스트 입력양식들에게 입력 힌트를 부여합니다.

필수값 지정 속성
```
<input type="text" required>
```
- required 속성을 사용하여 필수 입력 양식을 지정할 수 있습니다.

기본정보 그룹
```
<fieldset>
    <legend><h3>기본정보 그룹</h3></legend>
    입력양식1 : <input type="text">
</fieldset>
```
- fieldset 태그를 사용하여 form 태그 안 여러 입력양식들의 각 그룹을 가시적으로 나눌 수 있습니다.
- 이때 legend 태그를 사용하여 해당 그룹의 제목을 작성할 수 있습니다.

버튼의 종류
```
<button type="submit">등록</button>
<button type="reset">초기화</button>
<button type="button">일반버튼</button>  <!--자바스크립으로 내용설정-->
<input type="submit" value="회원가입">
<input type="reset" value="취소">
<input type="button" value="우편번호찾기">
```
- submit : form 안에 입력된 데이터를 처리하는 경로로 보내는 버튼입니다.
- reset : form 안에 입력된 데이터를 초기화하는 버튼입니다. 
- button : 초기 기능은 부여되어 있지 않으며 추후 개발자가 버튼에 자바스크립트로 기능을 할당하는 버튼입니다.
- 버튼들을 생성하는 태그는 button과 input이 있으며 type 속성으로 버튼의 종류를 지정합니다.
- button 태그는 시작종료 태그 사이에 버튼이름을, input 태그는 value 속성에 버튼이름을 지정합니다.

기타 속성들
- 텍스트관련 입력양식에 disabled를 선언하며 양식을 회색 빛으로 비활성화 합니다.(입력데이터 전송x)
- readonly를 선언하면 양식에 사용자가 값을 입력할 수 없게 읽기 전용이 됩니다.(입력데이터 전송o)
- 텍스트 입력양식의 초기값은 value 속성 값으로

### CSS 적용 방법
외부 스타일 시트
```
<head>
    <link href="외부스타일시트.css" type="text/css" rel="stylesheet">
</head>
```
```
<head>
    <style>
        @import url('외부스타일시트.css');
    </style>
<head>
```
- 공통 css를 한 곳에 두고 관리하기 용이하다. / css를 일차적으로 외부로 숨겨 공객하지 않을 수 있다.

내부 스타일 시트
```
<head>
    <style>
        body{background-color: beige;}
        div{background-color: pink;}
    </style>
</head>
```
- 내부 스타일 시트라고하며 head 태그 안에 style 태그를 선언하여 작성합니다.
- 이때 style 태그는 여러개 작성할 수 있으며 위에서부터 순서대로 적용합니다.
- 선택자(셀렉터)와 일치하는 본문의 모든 태그를 검색해 css를 적용합니다.
- 한 웹페이지 내에서 선택자와 일치하는 태그들의 디자인을 일관 적용할 수 있다.

인라인 스타일 시트
```
<body>
    <div style="background-color: lightgray; color: darkblue;">
</body>
```
- 인라인 스타일 시트라고하며 html 태그에 style 속성으로 속성명과 값들만 나열합니다.
- 모든 스타일시트 적용 방법 중에 우선순위가 가장 높습니다.
- 여러 태그들 중 하나의 태그에게만 직접 디자인할 수 있으나 코드관리가 불편해 사용 빈도가 낮다.

*내부스타일시트와 외부스타일시트가 중복되는 경우 우선순위는 아래 작성된 스타일시트가 최종적으로 적용되어 우선순위가 높다.

### 클래스 선택자와 아이디 선택자
클래스 선택자
```
<head>
    <style>
        .title{color: violet;}
    </style>
</head>
<body>
    <h1 class="title">클래스 선택자</h1>
    <div class="box content yellow">       // 클래스 속성값을 여러개 값는 태그
</body>
```
- html의 태그들 중 class 속성값이 일치하는 태그들을 검색합니다.
- 특정 태그에 종속된 class 선택자를 사용하고 싶다면 태그명.클래스값 으로 선택자를 사용합니다.

아이디 선택자
```
<head>
    <style>
        h3#title{color: skyblue;}
    </style>
</head>
<body>
    <h1 id="title">아이디 선택자를 배워보자</h1>
</body>
```
- 문법) #아이디속성값{속성들...}
- 태그명#아이디속성값{속성들...} / 특정태그에 종속적인 아이디 속성값 일치 태그 검색

*선택자들의 종류가 많아 겹치는 경우 선택자 별로 점술르 합산하여 점수가 높은 서택자 조합이 우선 적용됩니다.
- 태그선택자 : 1점
- 클래스선택자 : 10점
- 아이디선택자 : 100점 

### 후손, 자식, 형제 선택자
후손 선택자
```
<style>
    section h1{color: orange;}
</style>
```
- 선택자1과 선택자2 사이 공백을 사용하여 선택자1 하위에 일치하느 모든 선택자2 태그를 검색합니다. 
- 문법) 선택자1 선택자2{속성들...}
- 선택자1과 일치하는 태그의 하위에 선언된 모든 태그들 중 선택지2와 일치하는 태그들 검색
- 선택자1과 선택자2는 아이디, 클래스 선택자 등 다양한 선택자를 사용할 수 있습니다.

자식 선택자
```
<style>
    .parent>h1{color: skyblue;}
</style>
```
- 선택자1의 바로 아래 중첩된 태그 범위를 의미합니다.
- 문법) 선택자1>선택자2{속성들...}
- 선택자1의 자식태그들 중 선택자2와 일치하는 태그 검색
- 선택자1과 선택자2는 아이디, 클래스 선택자 등 다양한 선택자 조합이 이루어질 수 있다.

인접 형제 선택자
```
<style>
    h2+p{color: skyblue;}
</style>
```
- 같은 부모를 둔 태그를 기준으로 바로 아래 선언된 동생 태그를 의미합니다.
- '+' 기호를 사용하여 선택자1과 선택자2를 조합합니다.
- 문법) 선택자1+선택자2{속성들...}
- 선택자1 바로 아래 선언된 동생 태그가 선택자2와 일치하는 경우 검색

형제 선택자
```
<style>
    h2~p{color: skyblue;}
</style>
```
- 같은 부모를 둔 태그들 중 기준 태그의 아래 동생들 태그 전부를 대상으로 한다.
- 선택자 사용 기호는 ~ 기호를 사용한다.
- 문법) 선택자1 ~ 선택자2{속성들...}
- 선택자1과 일치하는 태그의 동생 태그들 중 선택자2와 일치하는 모든 태그 검색

전체 선택자
```
<style>
    *{border: 1px solid orange;}
</style>
```
- *기호를 사용합니다.
- 문법) *{속성들...} -> html 포함 본문의 모든 태그 검색
