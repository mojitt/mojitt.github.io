## 파이썬에서 객체 캡슐화
- 파이썬도 객체지향이기 때문에 객체 필드, 메소드의 캡슐화가 가능하기는 합니다.
- 자바처럼 public ~ private 접근제한자는 없으나 약식으로 캡슐화를 진행 할 수 있습니다.
- '_필드명' 으로 선언하면 개발자에게 외부 사용을 되도록 하지 말라는 약속으로 선언할 수 있습니다.
- *약속은 무시 가능 (쓰면 써짐)
- '__필드명' 으로 선언하면 내부적으로 필드의 이름을 바꿔 가지므로 외부에서 접근 할 수 없습니다.
- '__필드명' dms '_클래스명__필드명'으로 이름을 변환하여 사용 되어집니다.
- '__' 키워드는 캡슐화가 아니라 상속시 필드명 중복을 방지하기 위하여 사용하는 용도입니다.

```
class Sample:
    def __init__(self):
        self._field1 = '캡슐화 필드1'
        self.__field2 = '캡슐화 필드2'

sample1 = Sample()
print(sample1._field1)
#print(sample1.__field2)
print(sample1._Sample__field2)
```


## 모듈화
- 파이썬은 실행파일 하나와 여러 모듈로 하나의 프로그램이 개발됩니다.

#### 모듈이란?
- 클래스, 함수, 변수들이 모여있는 파이썬 파일 하나를 의미합니다.
- 실행파일에서 호출하여 사용하기 위한 목적의 파이썬 파일 하나 입니다.
- 실행파일에서 import 하여 사용할 수 있습니다.

#### 패키지 모듈화
- import 시 패키지 단위로는 import 할 수 없습니다.
- import 하고자하는 패키지의 __init__.py는 패키지 import시 최초 한번 실행되는 초기화 파일로 해당 파일에서 import 하고자하는 모듈들을 초기화 할 수 있습니다.
- 패키지명으로 import 하면 __init__.py를 import 하는 것과 같으므로 __init__.py에 필요한 모듈을 import 해두면 패키지 import 만으로 하위 모듈을 사용할 수 있습니다.
- 패키지 import 시 * 기호를 사용하면 __inti__.py에 있는 모든 요소를 불러와 사용할 수 있습니다.
- __init__.py에서 __all__ 변수를 사용하면 * import 시 포함 될 요소들을 제한할 수 있습니다.

```
#main.py

# echo.py import
#import sound.echo

from sound import *
import graphic


echo.echo_test()
print(VERSION)
#print(_SOUND) # _로 시작하는 요소들은 import의 대상이 되지 않습니다.

graphic.render.render_test()
```

## 예외처리
파이썬에서의 예외처리
- 실행시 예외가 발생하면 발생한 지점에서 프로그램이 강제 종료 됩니다.
- 예외가 발생하더라도 강제 죵료 되지 않게 하기위해서는 예외처리 코드가 필요합니다.

<pre>
예외처리 기본 문법
try:
    실행문1
    실행문2
    ...
    실행문n
except:
    예외처리문1
    ...
    예외처리문n
</pre>

- try문에서 예외 발생 지점 아래 실행문들은 실행되지 않고 except 구문으로 넘어갑니다.

```
#print(10/0) # 이 지점에서 예외가 발생하며 프로그램 강제 종료 됨

try:
    print('예외 발생 전')
    print(10/0)
    print('예외 발생 후')
except:
    print('예외발생!!')
    
print('-'*20)
# except 예외이름: -> 해당 이름의 예외만 처리할 수 있습니다.
try:
    print(10/0)
except ZeroDivisionError:
    print("예외발생!!")
    
print('-'*20)
# except 예외이름 as 변수명: print(변수명) -> 발생 예외 메시지를 확인할 수 있습니다.
try:
    list1 = [1,2,3]
    print(list1[3])
except IndexError as e:
    print(e)
   
print('-'*20)
# 모든 예외를 걸러 낼 때에는 예외 메시지는 Exception으로 처리하면 됩니다.
try:
    print(10/0) 
except Exception as e:
    print(e)

print('-'*20)    
# finally는 예외가 발생해도 안해도 반드시 실행되는 구문
try:
    print('예외처리 중')
    print(10/0)
except:
    print('예외발생')
finally:
    print('예외처리 종료')
    
print('-'*20)
# else문을 사용하여 try에서 오류가 발생하지 않았을 경우에 대한 실행문을 작성할 수 있습니다.
try:
    print('예외처리 중')
except:
    print('예외발생2!!')
else:
    print('정상종료')
finally:
    print('무조건 호출')
    
print('-'*20)
# 여러개의 예외를 별도 처리를 할 때에는 다중예외처리를 사용합니다.
# 상위 예외를 먼저 작성하면 오류는 안나지만 별도 처리는 할 수 없습니다.
try:
    value = int(input('숫자를 입력하세요:'))
    print(value/0)
except ZeroDivisionError as e:
    print(e)
except Exception as e:
    print(e)

print('-'*20)
# 만약 발생한 여러 예외에 대하여 처리하고자 하는 실행문이 같다면 멀티 예외처리로 작성할 수 있습니다.
# except (예외명, ..., 예외명) [as 변수명]:
try:
    value = int(input('숫자를 입력하세요:'))
    print(value/0)
except (ZeroDivisionError, ValueError) as e:
    print(e)
    print('제로 또는 값 오류1')
except Exception as e:
    print(e)
    print('다른 예외')


print('-'*40)

scores = []

try:
    count = 1
    for i in range(1,11):
        score = int(input('점수를 입력하세요.'))
        scores[count] = score
        count += 1        
except ValueError as e:
    print(e)
except IndexError as e:
    print(e)
except KeyboardInterrupt as e:
    print(e)
print(scores)

print('프로그램 종료')
```

#### 오류 만들기
- 사용자 정의 오류 클래스를 선언할 때 Exception을 상속 받으면 예외 클래스가 될 수 있습니다.
- __str__ 메소드를 재정의하면 예외 메시지를 수정할 수 있습니다.
- 사용자 정의 오류는 예외 발생 조건일 때 강제로 예외를 발생하여 호출됩니다.

```
class NullPointerException(Exception):
    def __str__(self):
        return '문자열은 null을 가질 수 없습니다.' # 오버라이딩

str1 = 'null'

try:
    if str1 == 'null':
        raise NullPointerException
except NullPointerException as e:
    print(e)
    
print('-'*40)

class Limit100Error(Exception):
    def __str__(self):
        return '합은 100을 넘길 수 없습니다.'

try:
    num1 = int(input('숫자1 입력하세요: '))
    num2 = int(input('숫자2 입력하세요: '))
    result = num1+num2
    if result > 100:
        raise Limit100Error
    print(result)
except Exception as e:
    print(e)
        

print('프로그램 종료')
```

#### 9. filter(함수, 반복객체)
- 반복객체의 원소들 중 함수의 호출 결과가 True인 원소들 집합을 반환합니다.
- 호출 결과는 filter 객체이므로 list() 또는 원하는 반복객체 타입으로 변환하여 사용해야합니다.

```
list4 = [1,2,3,4,5]

def even_func(num):
    return num%2 == 0

result = list(filter(even_func, list4))
print(result)

result = list(filter(lambda num: num%2 == 1, list4))
print(result)
```
