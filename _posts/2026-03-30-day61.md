## Promise
#### Promise란?
- 콜백 함수의 중첩(콜백 헬)으로 인한 가독성 저하와 에러 처리 어려움을 해결하기 위해 ES6에서 도입된 비동기 처리 전용 객체.
- 비동기 작업이 언제 완료되는지 알 수 없어도, 완료된 뒤의 동작을 .then() / .catch()로 미리 등록해 둘 수 있음.

#### Promise의 3가지 상태
- Pending 대기 — 비동기 처리가 아직 완료되지 않은 초기 상태
- Fulfilled 이행 — 처리 성공, resolve(값) 호출됨 → .then() 실행
- Rejected 실패 — 처리 실패, reject(에러) 호출됨 → .catch() 실행

#### 핵심 규칙
- .then() 안에서 값을 return 하면 다음 .then()으로 전달됨.
- .then() 안에서 새 Promise를 return 하면 그 Promise가 완료될 때까지 기다림.
- 어디서든 에러가 throw 되면 가장 가까운 .catch()로 점프함.

[ 콜백 방식 vs Promise 방식 비교 ]

| 구분 | 콜백 방식 | Promise 방식 |
|---|---|---|
| 중첩 구조 | 콜백 안에 콜백 → 콜백 헬 | `.then().then()` 체이닝으로 평탄화 |
| 에러 처리 | 각 콜백마다 개별 처리 필요 | 마지막 `.catch()` 한 곳에서 통합 처리 |
| 가독성 | 들여쓰기 지옥 | 선형 흐름으로 읽기 쉬움 |

## fetch
#### fetch()란?
- 브라우저 내장 함수로, 서버에 HTTP 요청을 보내고 응답을 Promise로 반환.
- 별도 라이브러리 없이 사용 가능. 옵션을 지정하지 않으면 기본값은 GET 방식.
- 문법: fetch(url [, options])

#### 응답 처리 2단계
- 1단계 — 응답 헤더 수신 즉시 Promise 이행. response.ok(200~299이면 true) / response.status(실제 코드)로 성공 여부 확인.
- 2단계 — 추가 메서드를 호출해 응답 본문(body)을 읽음.

- ⚠️ fetch는 404, 500 응답도 reject하지 않음. 반드시 response.ok를 직접 체크해야 함.

[ response 본문 읽기 메서드 ]

| 메서드 | 반환 타입 | 사용 예 |
|---|---|---|
| `response.json()` | JSON → 객체 | REST API, JSON 파일 응답 (가장 자주 사용) |
| `response.text()` | 문자열 | HTML, 텍스트 파일 응답 |
| `response.blob()` | Blob (바이너리) | 이미지, 파일 다운로드 |
| `response.formData()` | FormData | `multipart/form-data` 응답 |
| `response.arrayBuffer()` | ArrayBuffer | 오디오, 바이너리 로우 데이터 |

## async / await
#### async/await란?
- Promise를 더 읽기 쉽게 작성하기 위한 ES8(ES2017) 문법.
- 비동기 코드를 마치 동기 코드처럼 위에서 아래로 읽을 수 있게 해줌.

- async 함수: 함수 앞에 async를 붙이면 반환값이 자동으로 Promise로 래핑됨.
- await: async 함수 내부에서만 사용 가능. Promise가 완료될 때까지 해당 줄에서 대기.

- 에러 처리: try / catch로 동기 코드와 동일한 방식으로 처리 가능.

[ .then() 체이닝 vs async/await 비교 ]

| 구분 | `.then()` 체이닝 | `async/await` |
|---|:---:|:---:|
| 가독성 | 체이닝 깊어질수록 복잡 | 동기 코드처럼 선형으로 읽힘 |
| 에러 처리 | `.catch()` | `try / catch` 사용 가능 |
| 반환값 | Promise 반환 | `async` 함수도 Promise 반환 (동일) |
| `await` 사용 | 사용 불가 | `async` 함수 내부에서만 사용 가능 |

## axios
#### axios란?
- 브라우저와 Node.js 모두에서 동작하는 Promise 기반 HTTP 클라이언트 라이브러리.
- fetch의 불편함(response.ok 수동 체크, JSON 수동 파싱, 요청 헤더 수동 설정 등)을 자동으로 처리해줌.
- 실무에서 fetch보다 axios를 훨씬 많이 사용하므로 반드시 익혀두어야 함.

#### 설치
- CDN: <script src="https://cdnjs.cloudflare.com/ajax/libs/axios/1.6.7/axios.min.js"></script>
- npm: npm install axios (React, Vue 등 프레임워크 환경)

[ fetch vs axios vs $.ajax 비교 ]

| 구분 | fetch | axios | $.ajax |
|---|:---:|:---:|:---:|
| 라이브러리 필요 | 브라우저 내장 | 별도 설치 | jQuery 필요 |
| Node.js 지원 | Node 18+ 내장 | 완전 지원 | 브라우저 전용 |
| JSON 자동 파싱 | `.json()` 수동 호출 | `response.data`로 바로 접근 | `dataType:'json'` 설정 시 |
| 에러 처리 | `response.ok` 수동 체크 | 4xx/5xx 자동 reject | `.fail()` 자동 호출 |
| 요청 취소 | `AbortController` 직접 구현 | `CancelToken` / `AbortController` | `jqXHR.abort()` |
| 인터셉터 | 직접 구현 | 요청/응답 인터셉터 내장 | 직접 구현 |
| 기본값 설정 | 매번 설정 | `axios.defaults` / `instance` | `$.ajaxSetup()` |
| 실무 사용 | 간단한 요청 | 실무 표준 | 레거시 jQuery 프로젝트 |

## jQuery — $.ajax()
#### $.ajax()란?
- jQuery가 제공하는 비동기 HTTP 통신 메서드.
- 옵션이 직관적이고, .done / .fail / .always 체이닝으로 성공·실패·공통 처리를 분리할 수 있음.
- 현재는 axios 또는 fetch + async/await가 표준이나, 레거시 jQuery 프로젝트에서 자주 접하므로 반드시 읽고 이해할 수 있어야 함.

- 문법: $.ajax({ 옵션 }) 또는 $.ajax({ 옵션 }).done().fail().always()

[ $.ajax() 주요 옵션 ]

| 옵션 | 기본값 | 설명 |
|---|:---:|---|
| `url` | ― | 요청을 보낼 서버 주소 |
| `method` | `'GET'` | HTTP 요청 방식 (GET / POST / PUT / DELETE) |
| `data` | ― | 서버에 전달할 파라미터 `{ 'key': value }` — 자동으로 쿼리스트링 변환 |
| `dataType` | `'json'` | 응답 데이터 타입 (json / text / html / xml) |
| `contentType` | `'application/x-www-form-urlencoded'` | 요청 본문 타입. JSON 전송 시 `'application/json'` 으로 변경 |
| `success` | ― | 요청 성공 시 콜백 `function(data){}` |
| `error` | ― | 요청 실패 시 콜백 `function(jqXHR, textStatus, error){}` |
| `timeout` | ― | 요청 타임아웃(ms). 초과 시 fail 호출 |

[ 메서드 체이닝 — .done() / .fail() / .always() ]

| 메서드 | 실행 조건 | 콜백 인수 |
|---|:---:|---|
| `.done(fn)` | HTTP 요청 성공 시 | `(data, textStatus, jqXHR)` |
| `.fail(fn)` | HTTP 요청 실패 시 | `(jqXHR, textStatus, errorThrown)` |
| `.always(fn)` | 성공·실패 무관하게 항상 실행 | 성공: `(data, textStatus, jqXHR)` / 실패: `(jqXHR, textStatus, errorThrown)` |

```
/*
    * jqXHR : XMLHttpRequest 객체 (ajax 요청에 대한 정보, state, statusText 등...)
    * textStatus : 'success', 'error', 'abort', ... 등
    * error : error message
    */
$.ajax({
    url : './js/memberInfo.json',
    method : 'get',
    dataType : 'json',
    success : function(data, textStatus, jqXHR){
        console.log(data, textStatus, jqXHR);
    },
    error : function(jqXHR, textStatus, error){
        console.log(jqXHR, textStatus, error);
    }
});

$('#jqAjaxChain').click(e => {
    // 체이닝 기법
    const request = $.ajax({
        url : './js/memberInfo1.json',
        method : 'get',
        dataType : 'json',
        async : false // 동기식 ajax 요청
    });

    // 성공시 .done()
    request.done((data, textStatus, jqXHR) => {
        $('#ajaxResult').text(JSON.stringify(data, null, 2));
    });

    // 실패시 .fail()
    request.fail((jqXHR, textStatus, error) => {
        $('#ajaxResult').text(JSON.stringify(jqXHR, null, 2));
    });

    // 성공/실패시 모두 실행 : .always()
    request.always((jqXHR, textStatus, error) => {
        console.log('모두 실행');
    });
});

$('#jqAjaxOption').click(function(){

            $('#ajaxResult').text(JSON.stringify(jqXHR, null, 2));

    });
});
```

## 공공 API 활용 — 기상청 단기예보
#### 공공 API 사용 방법
- 공공데이터포털(https://www.data.go.kr)에서 API 키(serviceKey)를 발급받아 사용.
- 요청 시 data 옵션에 파라미터를 객체로 전달하면 jQuery가 자동으로 쿼리스트링으로 변환.

#### moment.js: 날짜·시간 포맷을 쉽게 다루는 유틸리티 라이브러리.
- moment().format('YYYYMMDD') → 오늘 날짜를 "20250414" 형식으로 반환.
- moment().add(1, 'h') → 현재 시각 + 1시간.
- moment().subtract(1, 'days') → 현재 날짜 - 1일.

- ⚠️ API Key는 소스코드에 직접 노출하지 말 것. 실제 서비스에서는 서버에서 관리하거나 환경변수로 분리.

[ 기상청 단기예보 API 주요 파라미터 ]

| 파라미터 | 설명 | 예시 |
|---|---|:---:|
| `serviceKey` | API 인증키 (공공데이터포털 발급) | URL 인코딩된 키 문자열 |
| `base_date` | 예보 기준 날짜 | `20250414` |
| `base_time` | 예보 기준 시간 (1일 8회 제공) | `0200` / `0500` / `0800` / `1100` / `1400` / `1700` / `2000` / `2300` |
| `nx` / `ny` | 예보 지점 격자 좌표 | 전주: nx=63, ny=89 |
| `dataType` | 응답 형식 | `JSON` |
