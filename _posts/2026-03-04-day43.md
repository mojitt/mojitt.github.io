#### 생성자 함수
생성자 함수는 객체를 생성하기 위한 템플릿 역할을 하는 함수.

특징:
- new 연산자와 함께 사용
- 함수명은 파스칼 표기법 (PascalCase) 사용
- this 키워드로 속성과 메서드 정의
- 같은 구조의 객체를 반복적으로 생성 가능

주의사항
- this로 정의한 멤버: 외부 접근 가능 (public)
- let/const로 정의한 변수: 외부 접근 불가 (private)
- 화살표 함수는 생성자로 사용 불가

| 구분       | 생성자 함수          | 클래스           |
|------------|---------------------|------------------|
| 문법       | function 키워드     | class 키워드     |
| 호이스팅   | 가능                | 불가능 (TDZ)     |
| private 속성 | 클로저 활용          | # 접두사         |
| 상속       | 프로토타입 체인      | extends 키워드   |
| 가독성     | 보통                | 우수             |
| 호환성     | 모든 브라우저        | ES6 이상         |

```
//생성자 함수
function User(){
    // property 속성 선언 (this.속성명) public
    //this = {}; // 생성자 함수에서 자동생성
    this.userName = '홍길동';

    // private
    let _userAge = 25;

    // method 메소드 선언 (this.메소드명 = function(){})
    this.getUserName = function(){
        return this.userName;
    }
    this.setUserName = function(userName){
        return this.userName = userName;
    }
    this.getUserAge = function(){
        return _userAge;
    }
    this.setUserAge = function(userAge){
        _userAge = userAge;
    }
    //return this; // 생성자 함수에서 자동생성
}
const user1 = new User();
console.log(user1);
```

#### 프로토타입 (Prototype)
프로토타입은 생성자 함수의 원형 객체.

장점
- 메모리 효율성: 메서드를 한 번만 생성하여 모든 인스턴스가 공유
- 동적 확장: 생성 후에도 메서드 추가 가능
- 상속 구현 가능

주의
- 내장 객체의 프로토타입을 확장하는 것은 신중하게 사용해야 합니다.
- 다른 라이브러리와 충돌할 수 있으며, 예상치 못한 동작을 일으킬 수 있습니다.

```
const Member = (() => {
    let _pw;

    function Member(id, pw) {
        this.id = id;    
        _pw = pw;
        this.getId = function(){
            return this.id;
        }
    }
    // 원형 객체 확장 prototype
    Member.prototype.setId = function(id){
        this.id = id;
    }
    Member.prototype.getPw = function(){
        return _pw;
    }
    Member.prototype.setPw = function(pw){
        _pw = pw;
    }
    return Member;
})();

const member1 = new Member('ksmartid1', 'ksamrtpw1');
console.log(member1);
console.log(member1.getPw());
const member2 = new Member('ksmartid2', 'ksamrtpw2');
console.log(member2);
console.log(member2.getPw());

Array.prototype.numTotal = function(){
    let total = 0;
    for(const element of this){
        total += element;
    }
    return total;
}
const protoArr = [100,200,300];
console.log(protoArr.numTotal()); 
```

#### 클래스 상속 (Inheritance)
- extends 키워드를 사용하여 클래스를 상속받을 수 있음.
- 자식 클래스는 부모 클래스의 속성과 메서드를 물려받음.

상속 장점
- 코드 재사용성 향상
- 계층적 구조 표현 가능
- 유지보수 용이

상속 단점
- 상속 구조가 복잡해질 수 있음
- 다중 상속 불가
- 부모 클래스 변경 시 자식 클래스에 영향

```
class Course {
    // 부모클래스 : 과정
    name = '기업요구 ICT 과정';
    getName(){
        return this.name;
    }
    setName(name){
        this.name = name;
    }
}
// 자식클래스 : 회차
class Round extends Course {
    constructor(gisu){
        super();
        this.gisu = gisu;
    }

    getGisu(){
        return this.gisu;
    }
    setGisu(gisu){
        this.gisu = gisu;
    }

    // 메소드 오버라이딩
    getName(){
        return `${super.getName()} 기수: ${this.gisu}`;
    }
}
const round58 = new Round(58);
console.log(round58.getName());

// 좌항 instanceof 우항 : 우항에 위치한 원형객체타입을 인스턴스가 생성됐는지 여부 확인
console.log(round58 instanceof Round); // true
console.log(round58 instanceof Course); // true
console.log(round58 instanceof Array); // false
```

#### 모듈패턴
- 코드를 독립된 모듈로 나누어 관리하고 재사용할 수 있게 해주는 디자인패턴
- 모듈 패턴을 이용해 기능을 내보낼 때는 export 키워드를 사용

```
// module1.js
export const fncA = () => {
    console.log('fncA');
}

export const fncB = () => {
    console.log('fncB');
}

export const msg = 'hello';

// module2.js
const fncC = () => {
    console.log('fncC');
}

const fncD = () => {
    console.log('fncD');
}

export {fncC, fncD}

// module3.js
const fncE = () => {
    console.log('fncE');
}

export default fncE;
/*
or
export default function fncE() {
    console.log('fncE');
}
*/

// 통합 : module.js
import { fncA, fncB, msg } from "./module1.js";
import { fncC, fncD } from "./module2.js";
import fncE from "./module3.js";

fncA();
fncB();
console.log(msg);
fncC();
fncD();
fncE();

// script 로드 시
<script type="module" src="js/module.js"></script>
```
