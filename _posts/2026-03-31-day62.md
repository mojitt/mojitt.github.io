## Spring Framework

#### Framework란?
특정 애플리케이션 개발을 위한 구조나 뼈대를 제공하는 도구  

| 특징 | 프레임워크 | 라이브러리 |
| :--- | :--- | :--- |
| 제어흐름 | 프레임워크 제어 (제어의 역전, IOC) | 개발자 제어 (개발자가 함수 호출) |
| 사용방식 | 미리 정의된 구조나 템플릿 문법에 준수함 | 개발자가 원하는 대로 기능을 호출하여 사용 |
| 유연성 | 프레임워크 규칙 구조를 준수하기에 감소 | 개발자가 필요에 따라 사용함으로 유연함 |
| 개발속도 | 구조나 템플릿 제공 (빠른 개발 시작)<br>자동화된 기능제공 | 모든 구조와 기능 개발자가 직접구현 (상대적 개발속도 느림) |

#### Spring Framework란?
자바 엔터프라이즈 애플리케이션을 개발하기 위한 모든 기능을 종합적으로 제공하는 경량화된 솔루션  

- IoC (Inversion of Control) : 제어의 역전
	- 모든 객체 제어권을 프레임워크에 위임
- DI (Dependency Injection) : 의존성 주입
	- 각 클래스 사이에 필요로 하는 의존관계를 빈 설정
	- 정보를 바탕으로 컨테이너가 자동으로 연결
- AOP (Aspect Oriented Programing)
	- 관심사별로 모듈화하여 코드 관리
- PSA (Portable Service Abstraction)
	- 다양한 서비스 인터페이스나 구현을 추상화하여 서로 
	- 다른 환경과 기술 스택 간에 이식 가능한 서비스를 제공

#### POJO (Plain Old Java Object) – 1 (느슨한 결합)
자바 객체가 특별한 제약없이, 단순히 getter, setter 메소드와 생성자만을 갖는 순수 자바 객체

```
public class UserService {
    private Database database;

    public UserService(Database database) {
        this.database = database; 
    }

    public void saveUser(User user) {
        database.save(user); 
    }
}

public interface Database { 
    void save(User user); 
}

public class Process {
    
    public static void main(String[] args) {

          Database database = new MySqlDatabase();
          UserService userService = new UserService(database);
          User user = new User();
          userService.saveUser(user);

    }
}

// UserService 클래스는 단순히 getter setter 생성자 메소드만 정의 되었으므로 데이터베이스 변경시 코드의 변경 없음.
```

#### POJO (Plain Old Java Object) – 2 (강결합)
자바 객체가 특별한 제약없이, 단순히 getter, setter 메소드와 생성자만을 갖는 순수 자바 객체

```
public class UserService {
    private MySQLDatabase mySQLDatabase;

    public UserService() {
        this.mySQLDatabase = new MySQLDatabase(); 
    }

    public void saveUser(User user) {
        mySQLDatabase.save(user); 
    }
}

public class MySQLDatabase {
    public void save(User user) {
        System.out.println("MySQL데이터베이스저장");
    }
}

public class Process {
    
    public static void main(String[] args) {

          UserService userService = new UserService();
          User user = new User();
          userService.saveUser(user);

    }
}

/*
UserService 클래스는 생성시 
MySQLDatabase 클래스를 필히 생성하므로 
database 변경시 UserService 클래스의 코드변경이
필요하므로 단위 테스트 진행하기 어려움
*/
```

#### IOC (Inversion of Control)
객체의 생성 및 의존성 관리를 개발자가 아닌 프레임워크가 제어권을 담당

- 객체(Bean) 생명주기 관리
	- 객체 생성, 초기화, 서비스, 소멸
- 의존성 주입
	- 객체 간의 의존관계 자동 연결
- AOP 지원
	- 로깅, 트랜잭션 관리 등
- 이벤트 처리 등등
	- 애플리케이션 내에 발생하는 이벤트 관리
	- 다양한 빈 정의 방식 제공

#### AOP (Aspect Oriented Programing)
관점 지향 프로그래밍, 반복되고 공통적으로 사용되는 부분을 분리함으로써 모듈성을 증가시키는 프로그래밍기법

- 비즈니스 로직 실행 전
	- @Before
- 비즈니스 로직 실행 후
	- @After
- 비즈니스 로직 실행 전과 후
	- @Around
- 비즈니스 로직 실행 결과가 정상적으로 반환된 후
	- @AfterReturning
- 비즈니스 로직 실행 시 예외가 발생된 후
	- @AfterThrowing

#### PSA (Portable Service Abstraction)
환경의 변화와 관련없이 일관된 방식의 기술로의 접근환경을 제공하는 추상화구조

#### DI (Dependency Injection) 
객체 간의 의존성을 관리, 객체가 필요로 하는 다른 객체를 외부에서 주입하는 방식 

**필드 주입**

```
public class UserService {

    @Autowired
    private UserRepository userRepository;

}
```

**세터 주입**

```
public class UserService {

    private UserRepository userRepository;

    @Autowired
    public void setUserRepository(UserRepository userRepository) {
	this.userRepository = userRepository;
    }
}
```

**생성자 주입 (권장)**

```
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {

        this.userRepository = userRepository;

    }
}
```

## Spring Boot
#### Spring Booot - 주요특징
Spring Framework를 기반으로 한 오프소스 자바프레임워크, 쉽게 빠르게 개발할 수 있도록 지원하는 플랫폼  

- Spring Framework를 기반으로 한 개발 플랫폼
	- Spring Framework기반 애플리케이션 개발시 복잡한 설정없이 빠르게 프로젝트를 시작 가능
- Spring Framework 의존성 관리
	- Spring Boot 프로젝트에 사용되는 라이브러리를 감지하고 호환성 맞는 설정을 자동으로 설정해줌
- Spring Framework 자동구성
	- 개발자가 필요한 구성요소만 명시하면 Spring Boot가 알아서 구성 및 설정하기 때문에 개발 속도 향상
- 단독실행 가능한 스프링 애플리케이션 개발 (내장서버)
	- Tomcat, Jetty 또는 Undertow를 직접 내장, 별도의 웹서버 설치없이 웹어플리케이션을 실행가능
- 미리 준비된 Stater 종속성 제공
	- Spring Boot 프로젝트는 다양한 종류의 스타터 패키지를 제공하여 필요한 기능을 쉽게 추가할 수 있음

#### Spring framework mvc 개발 단계
 1. 사용할 스프링 MVC, JPA, 하이버네이트의 버전을 결정 
 2. 모든 다른 레이어 연결 해주는 스프링 콘텍스트 설정 
 3. 스프링 MVC로 웹 레이어 설정
 4. 데이터 레이어에서 하이버네이트를 설정
 5. 단위테스트, 트랜잭션 전략, 로깅, 모니터링 방법 결정 구현
 6. 웹 애플리케이션 서버에 배포하는 방법 결정 구현

#### Spring Framework MVC
MVC 디자인 패턴을 기반으로 웹 애플리케이션 구축하는데 필용한 다양한 기능을 제공

- Model (데이터)
	- 애플리케이션의 데이터, 데이터에 대한 처리와 상태 관리
- View (화면)
	- 사용자 인터페이스(UI), 모델의 데이터 사용자 표시
- Controller (제어)
	- 사용자 요청 처리하고 Model을 업데이트하거나 View 선택 역할
	- 사용자의 상호작용을 처리 즉, 애플리케이션의 흐름 제어 및 관리
