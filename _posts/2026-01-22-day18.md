## 조인(JOIN)
조인이란?  
- 두 개 이상의 테이블을 서로 묶어서 하나의 결과 집합으로 만들어 내는 것

#### INNER JOIN (내부조인)
- 동등 조인, 이퀴조인(Equi-Join)  
- 두 개의 테이블에 있는 공통 열의 값이 같은 것을 결과 반환

조건 키워드
- ON : 열의 이름이 달라도 상관없음
- USING : 열의 이름이 동일해야 한다. (*로 전체를 검색할 때 중복된 열은 생략되서 검색된다.)

```
SELECT
	*
FROM
	TABLE1 A INNER JOIN TABLE2 B
	ON A.KEY = B.KEY
```

#### LEFT OUTER JOIN (외부조인)
- 조인문의 왼쪽에 명시한 테이블을 기준으로 두 개의 테이블에 있는 공통 열의 값이 같은 것을 결과와
왼쪽 테이블에만 있는 데이터를 포함하여 결과 반환  
- 오른쪽테이블의 결과와 일치하지 않을 경우의 오른쪽 테이블의 데이터는 NULL로 반환

```
SELECT
	*
FROM
	TABLE1 A LEFT JOIN TABLE2 B
	ON A.KEY = B.KEY
```

#### RIGHT OUTER JOIN (외부조인)
- 조인문의 오른쪽에 명시한 테이블을 기준으로 두 개의 테이블에 있는 공통 열의 값이 같은 것을 결과와
오른쪽 테이블에만 있는 데이터를 포함하여 결과 반환  
- 왼쪽테이블의 결과와 일치하지 않을 경우의 왼쪽 테이블의 데이터는 NULL로 반환

```
SELECT
	*
FROM
	TABLE1 A RIGHT JOIN TABLE2 B
	ON A.KEY = B.KEY
```

#### SELF JOIN (셀프조인)
- 관계된 테이블이 아닌 조회하려는 자기자신 테이블을 조인하여 결과를 반환
- 동일한 테이블을 조인 시 별칭을 사용하여야 함

```
SELECT
	*
FROM
	TABLE1 A JOIN TABLE1 B  // 같은 테이블 내의 키를 참조
	ON A.KEY = B.KEY
```

#### CROSS JOIN (상호조인)
- 한쪽 테이블의 모든 행들과 다른쪽 테이블의 모든 행을 묶어서 결과 반환 
- 주로 테스트로 사용할 많은 용량의 데이터를 생성할 때 쓰임
- mysql 기준 : join, inner join 조건이 없을 시 cross join과 결과가 동일

```
SELECT
	*
FROM
	TABLE1 A CROSS JOIN TABLE1 B  // 곱집합, 잘 안씀
```

#### NATURAL JOIN (내부조인)
- 동등 조인, 이퀴조인(Equi-Join) 
- 특정 집합의 같은 이름을 가진 컬럼 간의 조인 집합을 출력

```
SELECT
	*
FROM
	TABLE1 A NATURAL JOIN TABLE1 B  // 느려서 잘 안씀
```
